<li><b><a href="/wiki/Abstract_data_type" title="Abstract data type">Abstract data type</a></b> – (ADT), is a <a href="/wiki/Mathematical_model" title="Mathematical model">mathematical model</a> for <a href="/wiki/Data_type" title="Data type">data types</a>, where a data type is defined by its behavior (<a href="/wiki/Semantics_(computer_science)" title="Semantics (computer science)">semantics</a>) from the point of view of a <i>user</i> of the data, specifically in terms of possible values, possible operations on data of this type, and the behavior of these operations. This contrasts with <a href="/wiki/Data_structure" title="Data structure">data structures</a>, which are concrete representations of data, and are the point of view of an implementer, not a user.</li>
<li><b><a href="/wiki/Abstract_method" class="mw-redirect" title="Abstract method">Abstract method</a></b> – is one with only a <a href="/wiki/Method_signature" class="mw-redirect" title="Method signature">signature</a> and no <a href="/wiki/Method_body" class="mw-redirect" title="Method body">implementation body</a>. It is often used to specify that a subclass must provide an implementation of the method. Abstract methods are used to specify <a href="/wiki/Interface_(computing)" title="Interface (computing)">interfaces</a> in some computer languages.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup></li>
<li><b><a href="/wiki/Abstraction_(software_engineering)" class="mw-redirect" title="Abstraction (software engineering)">Abstraction</a></b> – In <a href="/wiki/Software_engineering" title="Software engineering">software engineering</a> and <a href="/wiki/Computer_science" title="Computer science">computer science</a>, abstraction is:
<ul><li>the process of removing physical, spatial, or temporal details<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">&#91;2&#93;</a></sup> or <a href="/wiki/Attribute_(computing)" title="Attribute (computing)">attributes</a> in the study of objects or <a href="/wiki/System" title="System">systems</a> in order to more closely attend to other details of interest;<sup id="cite_ref-abstraction_3-0" class="reference"><a href="#cite_note-abstraction-3">&#91;3&#93;</a></sup> it is also very similar in nature to the process of <a href="/wiki/Generalization" title="Generalization">generalization</a>;</li>
<li>the <a href="/wiki/Abstract_and_concrete" title="Abstract and concrete">abstract</a> <a href="/wiki/Concept" title="Concept">concept</a>-<a href="/wiki/Object_(philosophy)" title="Object (philosophy)">objects</a> which are created by keeping common features or attributes to various concrete objects or systems of study<sup id="cite_ref-abstraction_3-1" class="reference"><a href="#cite_note-abstraction-3">&#91;3&#93;</a></sup> — the result of the process of abstraction.</li></ul></li>
<li><b><a href="/wiki/Agent_architecture" title="Agent architecture">Agent architecture</a></b> – is a <a href="/wiki/Blueprint" title="Blueprint">blueprint</a> for <a href="/wiki/Software_agent" title="Software agent">software agents</a> and <a href="/wiki/Intelligent_control" title="Intelligent control">intelligent control</a> systems, depicting the arrangement of components. The architectures implemented by <a href="/wiki/Intelligent_agent" title="Intelligent agent">intelligent agents</a> are referred to as <a href="/wiki/Cognitive_architecture" title="Cognitive architecture">cognitive architectures</a>.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">&#91;4&#93;</a></sup></li>
<li><b><a href="/wiki/Agent-based_model" title="Agent-based model">Agent-based model</a></b> – (ABM), is a class of <a href="/wiki/Computational_models" class="mw-redirect" title="Computational models">computational models</a> for <a href="/wiki/Computer_simulation" title="Computer simulation">simulating</a> the actions and interactions of autonomous agents (both individual or collective entities such as organizations or groups) with a view to assessing their effects on the system as a whole. It combines elements of <a href="/wiki/Game_theory" title="Game theory">game theory</a>, <a href="/wiki/Complex_systems" class="mw-redirect" title="Complex systems">complex systems</a>, <a href="/wiki/Emergence" title="Emergence">emergence</a>, <a href="/wiki/Computational_sociology" title="Computational sociology">computational sociology</a>, <a href="/wiki/Multi-agent_system" title="Multi-agent system">multi-agent systems</a>, and <a href="/wiki/Evolutionary_programming" title="Evolutionary programming">evolutionary programming</a>. <a href="/wiki/Monte_Carlo_method" title="Monte Carlo method">Monte Carlo methods</a> are used to introduce randomness.</li>
<li><b><a href="/wiki/Aggregate_function" title="Aggregate function">Aggregate function</a></b> – In <a href="/wiki/Database_management" class="mw-redirect" title="Database management">database management</a> an aggregate function is a <a href="/wiki/Subroutine" title="Subroutine">function</a> where the values of multiple rows are grouped together to form a single value of more significant meaning or measurement such as a <a href="/wiki/Set_(computer_science)" class="mw-redirect" title="Set (computer science)">set</a>, a <a href="/wiki/Set_(computer_science)#Multiset" class="mw-redirect" title="Set (computer science)">bag</a> or a <a href="/wiki/List_(computing)" class="mw-redirect" title="List (computing)">list</a>.</li>
<li><b><a href="/wiki/Agile_software_development" title="Agile software development">Agile software development</a></b> – is an approach to <a href="/wiki/Software_development" title="Software development">software development</a> under which requirements and solutions evolve through the collaborative effort of <a href="/wiki/Self-organizing_communities" class="mw-redirect" title="Self-organizing communities">self-organizing</a> and <a href="/wiki/Cross-functional_team" title="Cross-functional team">cross-functional</a> teams and their <a href="/wiki/Customer" title="Customer">customer(s)</a>/<a href="/wiki/End_user" title="End user">end user(s)</a>.<sup id="cite_ref-Collier_2011_5-0" class="reference"><a href="#cite_note-Collier_2011-5">&#91;5&#93;</a></sup> It advocates adaptive planning, evolutionary development, early delivery, and <a href="/wiki/Continual_improvement_process" title="Continual improvement process">continual improvement</a>, and it encourages rapid and flexible response to change.<sup id="cite_ref-WhatIsAgile_6-0" class="reference"><a href="#cite_note-WhatIsAgile-6">&#91;6&#93;</a></sup></li>
<li><b><a href="/wiki/Algorithm" title="Algorithm">Algorithm</a></b> – is an unambiguous specification of how to solve a class of problems. Algorithms can perform <a href="/wiki/Calculation" title="Calculation">calculation</a>, <a href="/wiki/Data_processing" title="Data processing">data processing</a> and <a href="/wiki/Automated_reasoning" title="Automated reasoning">automated reasoning</a> tasks.</li>
<li><b><a href="/wiki/Algorithm_design" class="mw-redirect" title="Algorithm design">Algorithm design</a></b> – refers to a method or mathematical process for problem solving and engineering algorithms. The design of algorithms is part of many solution theories of <a href="/wiki/Operation_research" class="mw-redirect" title="Operation research">operation research</a>, such as <a href="/wiki/Dynamic_programming" title="Dynamic programming">dynamic programming</a> and <a href="/wiki/Divide_and_conquer_algorithm" class="mw-redirect" title="Divide and conquer algorithm">divide-and-conquer</a>. Techniques for designing and implementing algorithm designs are also called algorithm design patterns,<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">&#91;7&#93;</a></sup> such as the template method pattern and decorator pattern.</li>
<li><b><a href="/wiki/Algorithmic_efficiency" title="Algorithmic efficiency">Algorithmic efficiency</a></b> – is a property of an <a href="/wiki/Algorithm" title="Algorithm">algorithm</a> which relates to the number of <a href="/wiki/Computational_resource" title="Computational resource">computational resources</a> used by the algorithm. An algorithm must be <a href="/wiki/Analysis_of_algorithms" title="Analysis of algorithms">analyzed</a> to determine its resource usage, and the efficiency of an algorithm can be measured based on usage of different resources. Algorithmic efficiency can be thought of as analogous to engineering <a href="/wiki/Productivity" title="Productivity">productivity</a> for a repeating or continuous process.</li>
<li><b><a href="/wiki/ASCII" title="ASCII">ASCII</a></b> – abbreviated from <b>American Standard Code for Information Interchange</b>, is a <a href="/wiki/Character_encoding" title="Character encoding">character encoding</a> standard for electronic communication. ASCII codes represent text in computers, <a href="/wiki/Telecommunications_equipment" title="Telecommunications equipment">telecommunications equipment</a>, and other devices. Most modern character-encoding schemes are based on ASCII, although they support many additional characters.</li>
<li><b><a href="/wiki/Application_programming_interface" title="Application programming interface">Application programming interface</a></b> – (API) is a set of subroutine definitions, <a href="/wiki/Communication_protocols" class="mw-redirect" title="Communication protocols">communication protocols</a>, and tools for building software. In general terms, it is a set of clearly defined methods of communication among various components. A good API makes it easier to develop a <a href="/wiki/Computer_program" title="Computer program">computer program</a> by providing all the building blocks, which are then put together by the <a href="/wiki/Programmer" title="Programmer">programmer</a>.</li>
<li><b><a href="/wiki/Application_software" title="Application software">Application software</a></b> – (app or application for short) is <a href="/wiki/Computer_software" class="mw-redirect" title="Computer software">computer software</a> designed to perform a group of coordinated functions, tasks, or activities for the benefit of the user. Examples of an application include a <a href="/wiki/Word_processor" title="Word processor">word processor</a>, a <a href="/wiki/Spreadsheet" title="Spreadsheet">spreadsheet</a>, an <a href="/wiki/Accounting_software" title="Accounting software">accounting application</a>, a <a href="/wiki/Web_browser" title="Web browser">web browser</a>, a <a href="/wiki/Media_player_(software)" class="mw-redirect" title="Media player (software)">media player</a>, an aeronautical <a href="/wiki/Flight_simulator" title="Flight simulator">flight simulator</a>, a <a href="/wiki/Console_game" title="Console game">console game</a> or a <a href="/wiki/Raster_graphics_editor" title="Raster graphics editor">photo editor</a>. The <a href="/wiki/Collective_noun" title="Collective noun">collective noun</a> application software refers to all applications collectively.<sup id="cite_ref-8" class="reference"><a href="#cite_note-8">&#91;8&#93;</a></sup> This contrasts with <a href="/wiki/System_software" title="System software">system software</a>, which is mainly involved with running the computer.</li>
<li><b><a href="/wiki/Array_data_structure" title="Array data structure">Array data structure</a></b> – or simply an array, is a <a href="/wiki/Data_structure" title="Data structure">data structure</a> consisting of a collection of <i>elements</i> (<a href="/wiki/Value_(computer_science)" title="Value (computer science)">values</a> or <a href="/wiki/Variable_(programming)" class="mw-redirect" title="Variable (programming)">variables</a>), each identified by at least one <i>array index</i> or <i>key</i>. An array is stored such that the position of each element can be computed from its index <a href="/wiki/Tuple" title="Tuple">tuple</a> by a mathematical formula.<sup id="cite_ref-9" class="reference"><a href="#cite_note-9">&#91;9&#93;</a></sup><sup id="cite_ref-andres_10-0" class="reference"><a href="#cite_note-andres-10">&#91;10&#93;</a></sup><sup id="cite_ref-garcia_11-0" class="reference"><a href="#cite_note-garcia-11">&#91;11&#93;</a></sup> The simplest type of data structure is a linear array, also called one-dimensional array.</li>
<li><b><a href="/wiki/Artifact_(software_development)" title="Artifact (software development)">Artifact</a></b> – is one of many kinds of tangible by-products produced during the development of software. Some artifacts (e.g., <a href="/wiki/Use_case" title="Use case">use cases</a>, <a href="/wiki/Class_diagram" title="Class diagram">class diagrams</a>, and other <a href="/wiki/Unified_Modeling_Language" title="Unified Modeling Language">Unified Modeling Language</a> (UML) models, requirements and design documents) help describe the function, architecture, and design of software. Other artifacts are concerned with the process of development itself—such as project plans, business cases, and risk assessments.</li>
<li><b><a href="/wiki/Artificial_intelligence" title="Artificial intelligence">Artificial intelligence</a></b> – (AI), sometimes called <a href="/wiki/Machine" title="Machine">machine</a> intelligence, is <a href="/wiki/Intelligence" title="Intelligence">intelligence</a> demonstrated by <a href="/wiki/Machine" title="Machine">machines</a>, in contrast to the <b><a href="/wiki/Natural" class="mw-redirect" title="Natural">natural</a> intelligence</b> displayed by humans and other animals.  In <a href="/wiki/Computer_science" title="Computer science">computer science</a>  AI research is defined as the study of "<a href="/wiki/Intelligent_agent" title="Intelligent agent">intelligent agents</a>": any device that perceives its environment and takes actions that maximize its chance of successfully achieving its goals.<sup id="cite_ref-Definition_of_AI_12-0" class="reference"><a href="#cite_note-Definition_of_AI-12">&#91;12&#93;</a></sup> Colloquially, the term "artificial intelligence" is applied when a machine mimics "cognitive" functions that humans associate with other <a href="/wiki/Human_mind" class="mw-redirect" title="Human mind">human minds</a>, such as "learning" and "problem solving".<sup id="cite_ref-FOOTNOTERussellNorvig20092_13-0" class="reference"><a href="#cite_note-FOOTNOTERussellNorvig20092-13">&#91;13&#93;</a></sup></li>
<li><b><a href="/wiki/ASCII" title="ASCII">ASCII</a></b> – American Standard Code for Information Interchange.</li>
<li><b><a href="/wiki/Assertion_(software_development)" title="Assertion (software development)">Assertion</a></b> – In <a href="/wiki/Computer_programming" title="Computer programming">computer programming</a>, an assertion is a <a href="/wiki/Statement_(computer_science)" title="Statement (computer science)">statement</a> that a <a href="/wiki/Predicate_(mathematical_logic)" title="Predicate (mathematical logic)">predicate</a> (<a href="/wiki/Boolean-valued_function" title="Boolean-valued function">Boolean-valued function</a>, i.e. a true–false <a href="/wiki/Expression_(computer_science)" title="Expression (computer science)">expression</a>) is always true at that point in code execution.  It can help a programmer read the code, help a compiler compile it, or help the program detect its own defects. For the latter, some programs check assertions by actually evaluating the predicate as they run and if it is not in fact true, an assertion failure, the program considers itself to be broken and typically deliberately <a href="/wiki/Crash_(computing)" title="Crash (computing)">crashes</a> or throws an assertion failure <a href="/wiki/Exception_handling" title="Exception handling">exception</a>.</li>

<li><b><a href="/wiki/Automata_theory" title="Automata theory">Automata theory</a></b> – is the study of <a href="/wiki/Abstract_machine" title="Abstract machine">abstract machines</a> and <a href="/wiki/Automaton" title="Automaton">automata</a>, as well as the <a href="/wiki/Computational_problem" title="Computational problem">computational problems</a> that can be solved using them. It is a theory in <a href="/wiki/Theoretical_computer_science" title="Theoretical computer science">theoretical computer science</a> and <a href="/wiki/Discrete_mathematics" title="Discrete mathematics">discrete mathematics</a> (a subject of study in both <a href="/wiki/Mathematics" title="Mathematics">mathematics</a> and <a href="/wiki/Computer_science" title="Computer science">computer science</a>). The word <i>automata</i> (the plural of <i>automaton</i>) comes from the Greek word αὐτόματα, which means "self-acting".</li>
<li><b><a href="/wiki/Automated_reasoning" title="Automated reasoning">Automated reasoning</a></b> – is an area of <a href="/wiki/Computer_science" title="Computer science">computer science</a> and <a href="/wiki/Mathematical_logic" title="Mathematical logic">mathematical logic</a> dedicated to understanding different aspects of <a href="/wiki/Reasoning" class="mw-redirect" title="Reasoning">reasoning</a>. The study of automated reasoning helps produce <a href="/wiki/Computer_programs" class="mw-redirect" title="Computer programs">computer programs</a> that allow computers to reason completely, or nearly completely, automatically. Although automated reasoning is considered a sub-field of <a href="/wiki/Artificial_intelligence" title="Artificial intelligence">artificial intelligence</a>, it also has connections with <a href="/wiki/Theoretical_computer_science" title="Theoretical computer science">theoretical computer science</a>, and even <a href="/wiki/Philosophy" title="Philosophy">philosophy</a>.</li>
<li><b><a href="/wiki/Bandwidth_(computing)" title="Bandwidth (computing)">Bandwidth</a></b> – is the maximum rate of data transfer across a given path. Bandwidth may be characterized as <b>network bandwidth</b>,<sup id="cite_ref-14" class="reference"><a href="#cite_note-14">&#91;14&#93;</a></sup> <b>data bandwidth</b>,<sup id="cite_ref-15" class="reference"><a href="#cite_note-15">&#91;15&#93;</a></sup> or <b>digital bandwidth</b>.<sup id="cite_ref-16" class="reference"><a href="#cite_note-16">&#91;16&#93;</a></sup><sup id="cite_ref-17" class="reference"><a href="#cite_note-17">&#91;17&#93;</a></sup></li>
<li><b><a href="/wiki/Bayesian_programming" title="Bayesian programming">Bayesian programming</a></b> – is a formalism and a methodology for having a technique to specify <a href="/wiki/Probability_distribution" title="Probability distribution">probabilistic models</a> and solve problems when less than the necessary information is available.</li>
<li><b><a href="/wiki/Benchmark_(computing)" title="Benchmark (computing)">Benchmark</a></b> – is the act of running a <a href="/wiki/Computer_program" title="Computer program">computer program</a>, a set of programs, or other operations, in order to assess the relative <b>performance</b> of an object, normally by running a number of standard tests and trials against it.<sup id="cite_ref-18" class="reference"><a href="#cite_note-18">&#91;18&#93;</a></sup> The term <i>benchmark</i> is also commonly utilized for the purposes of elaborately designed benchmarking programs themselves.</li>
<li><b><a href="/wiki/Best,_worst_and_average_case" title="Best, worst and average case">Best, worst and average case</a></b> – <b>best</b>, <b>worst,</b> and <b>average cases</b> of a given <a href="/wiki/Algorithm" title="Algorithm">algorithm</a> express what the <a href="/wiki/Resource_(computer_science)" class="mw-redirect" title="Resource (computer science)">resource</a> usage is <i>at least</i>, <i>at most</i> and <i>on average</i>, respectively. Usually the resource being considered is running time, i.e. <a href="/wiki/Time_complexity" title="Time complexity">time complexity</a>, but it could also be memory or other resource.  Best case is the function which performs the minimum number of steps on input data of n elements.  Worst case is the function which performs the maximum number of steps on input data of size n. Average case is the function which performs an average number of steps on input data of n elements.</li>
<li><b><a href="/wiki/Big_data" title="Big data">Big data</a></b> – is a term used to refer to <a href="/wiki/Data_set" title="Data set">data sets</a> that are too large or complex for traditional <a href="/wiki/Data_processing" title="Data processing">data-processing</a> <a href="/wiki/Application_software" title="Application software">application software</a> to adequately deal with. Data with many cases (rows) offer greater <a href="/wiki/Statistical_power" class="mw-redirect" title="Statistical power">statistical power</a>, while data with higher complexity (more attributes or columns) may lead to a higher <a href="/wiki/False_discovery_rate" title="False discovery rate">false discovery rate</a>.<sup id="cite_ref-19" class="reference"><a href="#cite_note-19">&#91;19&#93;</a></sup></li>
<li><b><a href="/wiki/Big_O_notation" title="Big O notation">Big O notation</a></b> – is a mathematical notation that describes the <a href="/wiki/Asymptotic_analysis" title="Asymptotic analysis">limiting behavior</a> of a <a href="/wiki/Function_(mathematics)" title="Function (mathematics)">function</a> when the argument tends towards a particular value or infinity.  It is a member of a family of notations invented by <a href="/wiki/Paul_Gustav_Heinrich_Bachmann" title="Paul Gustav Heinrich Bachmann">Paul Bachmann</a>,<sup id="cite_ref-Bachmann_20-0" class="reference"><a href="#cite_note-Bachmann-20">&#91;20&#93;</a></sup> <a href="/wiki/Edmund_Landau" title="Edmund Landau">Edmund Landau</a>,<sup id="cite_ref-Landau_21-0" class="reference"><a href="#cite_note-Landau-21">&#91;21&#93;</a></sup> and others, collectively called <b>Bachmann–Landau notation</b> or <b>asymptotic notation</b>.</li>
<li><b><a href="/wiki/Binary_number" title="Binary number">Binary number</a></b> – In mathematics and <a href="/wiki/Digital_electronics" title="Digital electronics">digital electronics</a>, a <b>binary number</b> is a <a href="/wiki/Number" title="Number">number</a> expressed in the <b>base-2 numeral system</b> or <b>binary numeral system</b>, which uses only two symbols: typically <a href="/wiki/0_(number)" class="mw-redirect" title="0 (number)">0 (zero)</a> and <a href="/wiki/1_(number)" class="mw-redirect" title="1 (number)">1 (one)</a>.</li>
<li><b><a href="/wiki/Binary_search_algorithm" title="Binary search algorithm">Binary search algorithm</a></b> – <b>binary search</b>, also known as <b>half-interval search</b>,<sup id="cite_ref-Williams1976_22-0" class="reference"><a href="#cite_note-Williams1976-22">&#91;22&#93;</a></sup> <b>logarithmic search</b>,<sup id="cite_ref-FOOTNOTEKnuth1998§6.2.1_(&quot;Searching_an_ordered_table&quot;),_subsection_&quot;Binary_search&quot;_23-0" class="reference"><a href="#cite_note-FOOTNOTEKnuth1998§6.2.1_(&quot;Searching_an_ordered_table&quot;),_subsection_&quot;Binary_search&quot;-23">&#91;23&#93;</a></sup> or <b>binary chop</b>,<sup id="cite_ref-FOOTNOTEButterfieldNgondi201646_24-0" class="reference"><a href="#cite_note-FOOTNOTEButterfieldNgondi201646-24">&#91;24&#93;</a></sup> is a <a href="/wiki/Search_algorithm" title="Search algorithm">search algorithm</a> that finds the position of a target value within a <a href="/wiki/Sorted_array" title="Sorted array">sorted array</a>.<sup id="cite_ref-FOOTNOTECormenLeisersonRivestStein200939_25-0" class="reference"><a href="#cite_note-FOOTNOTECormenLeisersonRivestStein200939-25">&#91;25&#93;</a></sup><sup id="cite_ref-26" class="reference"><a href="#cite_note-26">&#91;26&#93;</a></sup></li>
<li><b><a href="/wiki/Binary_tree" title="Binary tree">Binary tree</a></b> – is a <a href="/wiki/Tree_structure" title="Tree structure">tree</a> <a href="/wiki/Data_structure" title="Data structure">data structure</a> in which each node has at most two <a href="/wiki/Child_node" class="mw-redirect" title="Child node">children</a>, which are referred to as the <i><span id="left_child">left child</span></i> and the <i><span id="right_child">right child</span></i>.  A <a href="/wiki/Recursive_definition" title="Recursive definition">recursive definition</a> using just <a href="/wiki/Set_theory" title="Set theory">set theory</a> notions is that a (non-empty) binary tree is a <a href="/wiki/Tuple" title="Tuple">tuple</a> (<i>L</i>, <i>S</i>, <i>R</i>), where <i>L</i> and <i>R</i> are binary trees or the <a href="/wiki/Empty_set" title="Empty set">empty set</a> and <i>S</i> is a <a href="/wiki/Singleton_set" class="mw-redirect" title="Singleton set">singleton set</a>.<sup id="cite_ref-GarnierTaylor2009_27-0" class="reference"><a href="#cite_note-GarnierTaylor2009-27">&#91;27&#93;</a></sup> Some authors allow the binary tree to be the empty set as well.<sup id="cite_ref-Skiena2009_28-0" class="reference"><a href="#cite_note-Skiena2009-28">&#91;28&#93;</a></sup></li>
<li><b><a href="/wiki/Bioinformatics" title="Bioinformatics">Bioinformatics</a></b> – is an <a href="/wiki/Interdisciplinary" class="mw-redirect" title="Interdisciplinary">interdisciplinary</a> field that develops methods and <a href="/wiki/Software_tool" class="mw-redirect" title="Software tool">software tools</a> for understanding <a href="/wiki/Biology" title="Biology">biological</a> data. As an interdisciplinary field of science, bioinformatics combines <a href="/wiki/Biology" title="Biology">biology</a>, <a href="/wiki/Computer_science" title="Computer science">computer science</a>, <a href="/wiki/Information_engineering_(field)" title="Information engineering (field)">information engineering</a>, <a href="/wiki/Mathematics" title="Mathematics">mathematics</a> and <a href="/wiki/Statistics" title="Statistics">statistics</a> to analyze and interpret biological data. Bioinformatics has been used for <i><a href="/wiki/In_silico" title="In silico">in silico</a></i> analyses of biological queries using mathematical and statistical techniques.</li>
<li><b><a href="/wiki/Bit" title="Bit">Bit</a></b> – (a <a href="/wiki/Portmanteau" title="Portmanteau">portmanteau</a> of <b>binary digit</b>)<sup id="cite_ref-Mackenzie_1980_29-0" class="reference"><a href="#cite_note-Mackenzie_1980-29">&#91;29&#93;</a></sup> is a <a href="/wiki/Units_of_information" title="Units of information">basic unit</a> of <a href="/wiki/Information" title="Information">information</a> used in <a href="/wiki/Computing" title="Computing">computing</a> and digital <a href="/wiki/Communication" title="Communication">communications</a>.  A <a href="/wiki/Binary_number" title="Binary number">binary</a> digit can have only one of two <a href="/wiki/Value_(computer_science)" title="Value (computer science)">values</a>, and may be physically represented with a two-state device. These state values are most commonly represented as either a <span style="white-space:nowrap">0<span style="margin-left:0.25em">or</span><span style="margin-left:0.25em">1</span></span>.</li>
<li><b><a href="/wiki/Bit_rate" title="Bit rate">Bit rate</a></b> – In <a href="/wiki/Telecommunications" class="mw-redirect" title="Telecommunications">telecommunications</a> and <a href="/wiki/Computing" title="Computing">computing</a>, <b>bit rate</b> (<b>bitrate</b> or as a variable <i>R</i>) is the number of <a href="/wiki/Bit" title="Bit">bits</a> that are conveyed or processed per unit of time.<sup id="cite_ref-30" class="reference"><a href="#cite_note-30">&#91;30&#93;</a></sup></li>
<li><b><a href="/wiki/Blacklist_(computing)" title="Blacklist (computing)">Blacklist</a></b> – In <a href="/wiki/Computing" title="Computing">computing</a>, a <b>blacklist</b> or <b>block list</b> is a basic <a href="/wiki/Access_control" title="Access control">access control</a> mechanism that allows through all elements (email addresses, users, passwords, <a href="/wiki/URLs" class="mw-redirect" title="URLs">URLs</a>, <a href="/wiki/IP_address" title="IP address">IP addresses</a>, <a href="/wiki/Domain_name" title="Domain name">domain names</a>, file <a href="/wiki/MD5_Hash" class="mw-redirect" title="MD5 Hash">hashes</a>, etc.), except those explicitly mentioned. Those items on the list are denied access. The opposite is a <a href="/wiki/Whitelist" class="mw-redirect" title="Whitelist">whitelist</a>, which means only items on the list are let through whatever gate is being used. A <a href="/wiki/Greylisting" title="Greylisting">greylist</a> contains items that are temporarily blocked (or temporarily allowed) until an additional step is performed.</li>
<li><b><a href="/wiki/BMP_file_format" title="BMP file format">BMP file format</a></b> –  also known as <b>bitmap image file</b> or <b>device independent bitmap (DIB) file format</b> or simply a <b>bitmap</b>, is a <a href="/wiki/Raster_graphics" title="Raster graphics">raster graphics</a> <a href="/wiki/Image_file_format" class="mw-redirect" title="Image file format">image file format</a> used to store <a href="/wiki/Bitmap" title="Bitmap">bitmap</a> <a href="/wiki/Digital_image" title="Digital image">digital images</a>, independently of the <a href="/wiki/Display_device" title="Display device">display device</a> (such as a <a href="/wiki/Graphics_adapter" class="mw-redirect" title="Graphics adapter">graphics adapter</a>), especially on <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a><sup id="cite_ref-bmp_31-0" class="reference"><a href="#cite_note-bmp-31">&#91;31&#93;</a></sup> and <a href="/wiki/OS/2" title="OS/2">OS/2</a><sup id="cite_ref-os2bmp_32-0" class="reference"><a href="#cite_note-os2bmp-32">&#91;32&#93;</a></sup> operating systems.</li>
<li><b><a href="/wiki/Boolean_data_type" title="Boolean data type">Boolean data type</a></b> – is a <a href="/wiki/Data_type" title="Data type">data type</a> that has one of two possible values (usually denoted <i>true</i> and <i>false</i>), intended to represent the two <a href="/wiki/Truth_value" title="Truth value">truth values</a> of <a href="/wiki/Logic" title="Logic">logic</a> and <a href="/wiki/Boolean_algebra" title="Boolean algebra">Boolean algebra</a>. It is named after <a href="/wiki/George_Boole" title="George Boole">George Boole</a>, who first defined an algebraic system of logic in the mid 19th century. The Boolean data type is primarily associated with <a href="/wiki/Conditional_(computer_programming)" title="Conditional (computer programming)">conditional</a> statements, which allow different actions by changing <a href="/wiki/Control_flow" title="Control flow">control flow</a> depending on whether a programmer-specified Boolean <i>condition</i> evaluates to true or false. It is a special case of a more general <i>logical data type (see <a href="/wiki/Probabilistic_logic" title="Probabilistic logic">probabilistic logic</a>)—</i>logic need not always be Boolean.</li>
<li><b><a href="/wiki/Boolean_expression" title="Boolean expression">Boolean expression</a></b> – is used <a href="/wiki/Expression_(programming)" class="mw-redirect" title="Expression (programming)">expression</a> in a <a href="/wiki/Programming_language" title="Programming language">programming language</a> that produces a <a href="/wiki/Boolean_value" class="mw-redirect" title="Boolean value">Boolean value</a> when evaluated, that is one of <b>true</b> or <b>false</b>. A Boolean expression may be composed of a combination of the Boolean constants <b>true</b> or <b>false</b>, <a href="/wiki/Boolean_data_type" title="Boolean data type">Boolean-typed</a> variables, Boolean-valued operators, and <a href="/wiki/Boolean-valued_function" title="Boolean-valued function">Boolean-valued functions</a>.<sup id="cite_ref-33" class="reference"><a href="#cite_note-33">&#91;33&#93;</a></sup></li>
<li><b><a href="/wiki/Boolean_algebra" title="Boolean algebra">Boolean algebra</a></b> – In <a href="/wiki/Mathematics" title="Mathematics">mathematics</a> and <a href="/wiki/Mathematical_logic" title="Mathematical logic">mathematical logic</a>, <b>Boolean algebra</b> is the branch of <a href="/wiki/Algebra" title="Algebra">algebra</a> in which the values of the <a href="/wiki/Variable_(mathematics)" title="Variable (mathematics)">variables</a> are the <a href="/wiki/Truth_value" title="Truth value">truth values</a> <i>true</i> and <i>false</i>, usually denoted 1 and 0 respectively. Instead of <a href="/wiki/Elementary_algebra" title="Elementary algebra">elementary algebra</a> where the values of the variables are numbers, and the prime operations are addition and multiplication, the main operations of Boolean algebra are the <a href="/wiki/Logical_conjunction" title="Logical conjunction">conjunction</a> <i>and</i> denoted as ∧, the <a href="/wiki/Logical_disjunction" title="Logical disjunction">disjunction</a> <i>or</i> denoted as ∨, and the <a href="/wiki/Negation" title="Negation">negation</a> <i>not</i> denoted as ¬. It is thus a formalism for describing logical relations in the same way that elementary algebra describes numeric relations.</li>
<li><b><a href="/wiki/Byte" title="Byte">Byte</a></b> – is a <a href="/wiki/Units_of_information" title="Units of information">unit of digital information</a> that most commonly consists of eight <a href="/wiki/Bit" title="Bit">bits</a>, representing a <a href="/wiki/Binary_number" title="Binary number">binary number</a>. Historically, the byte was the number of bits used to encode a single <a href="/wiki/Character_(computing)" title="Character (computing)">character</a> of text in a computer<sup id="cite_ref-Buchholz_1962_34-0" class="reference"><a href="#cite_note-Buchholz_1962-34">&#91;34&#93;</a></sup><sup id="cite_ref-Bemer_1959_35-0" class="reference"><a href="#cite_note-Bemer_1959-35">&#91;35&#93;</a></sup> and for this reason it is the smallest <a href="/wiki/Address_space" title="Address space">addressable</a> unit of <a href="/wiki/Computer_memory" title="Computer memory">memory</a> in many <a href="/wiki/Computer_architecture" title="Computer architecture">computer architectures</a>.</li>
<li><b><a href="/wiki/Booting" title="Booting">Booting</a></b>- is starting up a <a href="/wiki/Computer" title="Computer">computer</a> or <a href="/wiki/Computer_appliance" title="Computer appliance">computer appliance</a> until it can be used. It can be initiated by hardware such as a button press or by software command. After the power is switched on, the computer is relatively dumb and can read only part of its storage called <a href="/wiki/Read-only_memory" title="Read-only memory">read-only memory</a>. There, a small program is stored called <a href="/wiki/Firmware" title="Firmware">firmware</a>. It does <a href="/wiki/Power-on_self-test" title="Power-on self-test">power-on self-tests</a> and, most importantly, allows accessing other types of memory like a <a href="/wiki/Hard_disk" class="mw-redirect" title="Hard disk">hard disk</a> and <a href="/wiki/Main_memory" class="mw-redirect" title="Main memory">main memory</a>. The firmware loads bigger <a href="/wiki/Computer_program" title="Computer program">programs</a> into the computer's <a href="/wiki/Random-access_memory" title="Random-access memory">main memory</a> and runs it.</li>
<li><b><a href="/wiki/Callback_(computer_programming)" title="Callback (computer programming)">Callback</a></b> – also known as a "call-after<sup id="cite_ref-36" class="reference"><a href="#cite_note-36">&#91;36&#93;</a></sup> " function, is any <a href="/wiki/Executable_code" class="mw-redirect" title="Executable code">executable code</a> that is passed as an <a href="/wiki/Argument_(computer_science)" class="mw-redirect" title="Argument (computer science)">argument</a> to other code that is expected to <i>call back</i> (execute) the argument at a given time. This execution may be immediate as in a <b>synchronous callback</b>, or it might happen at a later time as in an <b>asynchronous callback</b>.</li>
<li><b><a href="/wiki/Central_processing_unit" title="Central processing unit">Central processing unit</a></b> – (<b>CPU</b>) is the <a href="/wiki/Electronic_circuit" title="Electronic circuit">electronic circuitry</a> within a <a href="/wiki/Computer" title="Computer">computer</a> that carries out the <a href="/wiki/Instruction_(computing)" class="mw-redirect" title="Instruction (computing)">instructions</a> of a <a href="/wiki/Computer_program" title="Computer program">computer program</a> by performing the basic <a href="/wiki/Arithmetic" title="Arithmetic">arithmetic</a>, logic, controlling and <a href="/wiki/Input/output" title="Input/output">input/output</a> (I/O) operations specified by the instructions. The computer industry has used the term "central processing unit" at least since the early 1960s.<sup id="cite_ref-weik1961_37-0" class="reference"><a href="#cite_note-weik1961-37">&#91;37&#93;</a></sup> Traditionally, the term "CPU" refers to a <b>processor</b>, more specifically to its processing unit and <a href="/wiki/Control_unit" title="Control unit">control unit</a> (CU), distinguishing these core elements of a computer from external components such as <a href="/wiki/Main_memory" class="mw-redirect" title="Main memory">main memory</a> and <a href="/wiki/I/O" class="mw-redirect" title="I/O">I/O</a> circuitry.<sup id="cite_ref-kuck_38-0" class="reference"><a href="#cite_note-kuck-38">&#91;38&#93;</a></sup></li>
<li><b><a href="/wiki/Character_(computing)" title="Character (computing)">Character</a></b> – In <a href="/wiki/Computer" title="Computer">computer</a> and machine-based <a href="/wiki/Telecommunication" title="Telecommunication">telecommunications</a> terminology, a <b>character</b> is a unit of <a href="/wiki/Information" title="Information">information</a> that roughly corresponds to a <a href="/wiki/Grapheme" title="Grapheme">grapheme</a>, grapheme-like unit, or symbol, such as in an <a href="/wiki/Alphabet" title="Alphabet">alphabet</a> or <a href="/wiki/Syllabary" title="Syllabary">syllabary</a> in the <a href="/wiki/Written_language" title="Written language">written</a> form of a <a href="/wiki/Natural_language" title="Natural language">natural language</a>.<sup id="cite_ref-39" class="reference"><a href="#cite_note-39">&#91;39&#93;</a></sup></li>
<li><b><a href="/wiki/Cipher" title="Cipher">Cipher</a></b> – In <a href="/wiki/Cryptography" title="Cryptography">cryptography</a>, a <b>cipher</b> (or <b>cypher</b>) is an <a href="/wiki/Algorithm" title="Algorithm">algorithm</a> for performing <a href="/wiki/Encryption" title="Encryption">encryption</a> or <a href="/wiki/Decryption" class="mw-redirect" title="Decryption">decryption</a>&#8212;a series of well-defined steps that can be followed as a procedure.</li>
<li><b><a href="/wiki/Class_(computer_science)" class="mw-redirect" title="Class (computer science)">Class</a></b> – In <a href="/wiki/Object-oriented_programming" title="Object-oriented programming">object-oriented programming</a>, a <b>class</b> is an extensible program-code-template for creating <a href="/wiki/Object_(object-oriented_programming)" class="mw-redirect" title="Object (object-oriented programming)">objects</a>, providing initial values for state (<a href="/wiki/Member_variable" title="Member variable">member variables</a>) and implementations of behavior (member functions or <a href="/wiki/Method_(computer_programming)" title="Method (computer programming)">methods</a>).<sup id="cite_ref-FOOTNOTEGammaHelmJohnsonVlissides199514_40-0" class="reference"><a href="#cite_note-FOOTNOTEGammaHelmJohnsonVlissides199514-40">&#91;40&#93;</a></sup><sup id="cite_ref-FOOTNOTEBruce20022.1_Objects,_classes,_and_object_types,_https&#58;//books.google.com/books?id=9NGWq3K1RwUC&amp;pg=PA18_41-0" class="reference"><a href="#cite_note-FOOTNOTEBruce20022.1_Objects,_classes,_and_object_types,_https://books.google.com/books?id=9NGWq3K1RwUC&amp;pg=PA18-41">&#91;41&#93;</a></sup> In many languages, the class name is used as the name for the class (the template itself), the name for the default <a href="/wiki/Constructor_(object-oriented_programming)" title="Constructor (object-oriented programming)">constructor</a> of the class (a <a href="/wiki/Subroutine" title="Subroutine">subroutine</a> that creates objects), and as the <a href="/wiki/Data_type" title="Data type">type</a> of objects generated by <a href="/wiki/Instance_(computer_science)" title="Instance (computer science)">instantiating</a> the class; these distinct concepts are easily conflated.<sup id="cite_ref-FOOTNOTEBruce20022.1_Objects,_classes,_and_object_types,_https&#58;//books.google.com/books?id=9NGWq3K1RwUC&amp;pg=PA18_41-1" class="reference"><a href="#cite_note-FOOTNOTEBruce20022.1_Objects,_classes,_and_object_types,_https://books.google.com/books?id=9NGWq3K1RwUC&amp;pg=PA18-41">&#91;41&#93;</a></sup></li>
<li><b><a href="/wiki/Class-based_programming" title="Class-based programming">Class-based programming</a></b> – or more commonly <b>class-orientation</b>, is a style of <a href="/wiki/Object-oriented_programming" title="Object-oriented programming">Object-oriented programming</a> (OOP) in which inheritance occurs via defining <i><a href="/wiki/Class_(computer_programming)" title="Class (computer programming)">classes</a></i> of <a href="/wiki/Object_(computer_science)" title="Object (computer science)">objects</a>, instead of inheritance occurring via the objects alone (compare <a href="/wiki/Prototype-based_programming" title="Prototype-based programming">prototype-based programming</a>).</li>
<li><b><a href="/wiki/Class-based_programming" title="Class-based programming">Class-orientation</a></b> – is a style of <a href="/wiki/Object-oriented_programming" title="Object-oriented programming">Object-oriented programming</a> (OOP) in which inheritance occurs via defining <i><a href="/wiki/Class_(computer_programming)" title="Class (computer programming)">classes</a></i> of <a href="/wiki/Object_(computer_science)" title="Object (computer science)">objects</a>, instead of inheritance occurring via the objects alone (compare <a href="/wiki/Prototype-based_programming" title="Prototype-based programming">prototype-based programming</a>).</li>
<li><b><a href="/wiki/Client_(computing)" title="Client (computing)">Client</a></b> – is a piece of <a href="/wiki/Computer_hardware" title="Computer hardware">computer hardware</a> or <a href="/wiki/Software" title="Software">software</a> that accesses a service made available by a <a href="/wiki/Server_(computing)" title="Server (computing)">server</a>. The server is often (but not always) on another <a href="/wiki/Computer_system" class="mw-redirect" title="Computer system">computer system</a>, in which case the client accesses the service by way of a <a href="/wiki/Computer_network" title="Computer network">network</a>.<sup id="cite_ref-42" class="reference"><a href="#cite_note-42">&#91;42&#93;</a></sup> The term applies to the role that programs or devices play in the <a href="/wiki/Client%E2%80%93server_model" title="Client–server model">client–server model</a>.</li>
<li><b><a href="/wiki/Cleanroom_software_engineering" title="Cleanroom software engineering">Cleanroom software engineering</a></b> – is a <a href="/wiki/Software_development_process" title="Software development process">software development process</a> intended to produce software with a certifiable level of <a href="/wiki/Reliability_engineering" title="Reliability engineering">reliability</a>. The cleanroom process was originally developed by <a href="/wiki/Harlan_Mills" title="Harlan Mills">Harlan Mills</a> and several of his colleagues including Alan Hevner at <a href="/wiki/IBM" title="IBM">IBM</a>.<sup id="cite_ref-43" class="reference"><a href="#cite_note-43">&#91;43&#93;</a></sup> The focus of the cleanroom process is on defect prevention, rather than defect removal.</li>
<li><b><a href="/wiki/Closure_(computer_programming)" title="Closure (computer programming)">Closure</a></b> – In <a href="/wiki/Programming_language" title="Programming language">programming languages</a>, a <b>closure</b> (also <b>lexical closure</b> or <b>function closure</b>) is a technique for implementing <a href="/wiki/Lexically_scoped" class="mw-redirect" title="Lexically scoped">lexically scoped</a> <a href="/wiki/Name_binding" title="Name binding">name binding</a> in a language with <a href="/wiki/First-class_function" title="First-class function">first-class functions</a>. <a href="/wiki/Operational_semantics" title="Operational semantics">Operationally</a>, a closure is a <a href="/wiki/Record_(computer_science)" title="Record (computer science)">record</a> storing a <a href="/wiki/Function_(computer_science)" class="mw-redirect" title="Function (computer science)">function</a><sup id="cite_ref-44" class="reference"><a href="#cite_note-44">&#91;a&#93;</a></sup> together with an environment.<sup id="cite_ref-45" class="reference"><a href="#cite_note-45">&#91;44&#93;</a></sup></li>
<li><b><a href="/wiki/Cloud_computing" title="Cloud computing">Cloud computing</a></b> – is shared pools of configurable computer <a href="/wiki/System_resource" title="System resource">system resources</a> and higher-level services that can be rapidly <a href="/wiki/Provisioning" class="mw-redirect" title="Provisioning">provisioned</a> with minimal management effort, often over the <a href="/wiki/Internet" title="Internet">Internet</a>.  Cloud computing relies on sharing of resources to achieve coherence and <a href="/wiki/Economies_of_scale" title="Economies of scale">economies of scale</a>, similar to a <a href="/wiki/Public_utility" title="Public utility">public utility</a>.</li>
<li><b><a href="/wiki/Library_(computing)" title="Library (computing)">Code library</a></b> – is a collection of <a href="/wiki/Non-volatile_memory" title="Non-volatile memory">non-volatile resources</a> used by <a href="/wiki/Computer_program" title="Computer program">computer programs</a>, often for <a href="/wiki/Software_development" title="Software development">software development</a>. These may include configuration data, documentation, help data, message templates, <a href="/wiki/Code_reuse" title="Code reuse">pre-written code</a> and <a href="/wiki/Subroutine" title="Subroutine">subroutines</a>, <a href="/wiki/Class_(computer_science)" class="mw-redirect" title="Class (computer science)">classes</a>, <a href="/wiki/Value_(computer_science)" title="Value (computer science)">values</a> or <a href="/wiki/Data_type" title="Data type">type</a> specifications. In <a href="/wiki/OS/360_and_successors" title="OS/360 and successors">IBM's OS/360 and its successors</a> they are referred to as <a href="/wiki/Partitioned_dataset" class="mw-redirect" title="Partitioned dataset">partitioned data sets</a>.</li>
<li><b><a href="/wiki/Computer_programming" title="Computer programming">Coding</a></b> – Computer programming is the process of designing and building an <a href="/wiki/Executable" title="Executable">executable</a> <a href="/wiki/Computer_program" title="Computer program">computer program</a> for accomplishing a specific <a href="/wiki/Computing" title="Computing">computing</a> task. Programming involves tasks such as analysis, generating <a href="/wiki/Algorithm" title="Algorithm">algorithms</a>, profiling algorithms' accuracy and resource consumption, and the implementation of algorithms in a chosen <a href="/wiki/Programming_language" title="Programming language">programming language</a> (commonly referred to as <b>coding</b><sup id="cite_ref-tumblr2014_46-0" class="reference"><a href="#cite_note-tumblr2014-46">&#91;45&#93;</a></sup><sup id="cite_ref-tumblr1_47-0" class="reference"><a href="#cite_note-tumblr1-47">&#91;46&#93;</a></sup>). The <a href="/wiki/Source_code" title="Source code">source code</a> of a program is written in one or more programming languages. The purpose of programming is to find a sequence of instructions that will automate the performance of a task for solving a given problem. The process of programming thus often requires expertise in several different subjects, including knowledge of the <a href="/wiki/Domain_(software_engineering)" title="Domain (software engineering)">application domain</a>, specialized algorithms, and formal <a href="/wiki/Logic" title="Logic">logic</a>.</li>
<li><b><a href="/wiki/Coding_theory" title="Coding theory">Coding theory</a></b> – is the study of the properties of <a href="/wiki/Code" title="Code">codes</a> and their respective fitness for specific applications. Codes are used for <a href="/wiki/Data_compression" title="Data compression">data compression</a>, <a href="/wiki/Cryptography" title="Cryptography">cryptography</a>, <a href="/wiki/Error_detection_and_correction" title="Error detection and correction">error detection and correction</a>, <a href="/wiki/Data_transmission" title="Data transmission">data transmission</a> and <a href="/wiki/Data_storage" title="Data storage">data storage</a>. Codes are studied by various scientific disciplines—such as <a href="/wiki/Information_theory" title="Information theory">information theory</a>, <a href="/wiki/Electrical_engineering" title="Electrical engineering">electrical engineering</a>,  <a href="/wiki/Mathematics" title="Mathematics">mathematics</a>, <a href="/wiki/Linguistics" title="Linguistics">linguistics</a>, and <a href="/wiki/Computer_science" title="Computer science">computer science</a>—for the purpose of designing efficient and reliable data transmission methods. This typically involves the removal of redundancy and the correction or detection of errors in the transmitted data.</li>
<li><b><a href="/wiki/Cognitive_science" title="Cognitive science">Cognitive science</a></b> – is the interdisciplinary, <a href="/wiki/Science" title="Science">scientific</a> study of the <a href="/wiki/Mind" title="Mind">mind</a> and its processes.<sup id="cite_ref-48" class="reference"><a href="#cite_note-48">&#91;47&#93;</a></sup>  It examines the nature, the tasks, and the functions of <a href="/wiki/Cognition" title="Cognition">cognition</a> (in a broad sense). Cognitive scientists study intelligence and behavior, with a focus on how nervous systems represent, process, and transform <a href="/wiki/Information" title="Information">information</a>. Mental faculties of concern to cognitive scientists include <a href="/wiki/Language" title="Language">language</a>, <a href="/wiki/Perception" title="Perception">perception</a>, <a href="/wiki/Memory" title="Memory">memory</a>, <a href="/wiki/Attention" title="Attention">attention</a>, <a href="/wiki/Reasoning" class="mw-redirect" title="Reasoning">reasoning</a>, and <a href="/wiki/Emotion" title="Emotion">emotion</a>; to understand these faculties, cognitive scientists borrow from fields such as <a href="/wiki/Linguistics" title="Linguistics">linguistics</a>, <a href="/wiki/Psychology" title="Psychology">psychology</a>, <a href="/wiki/Artificial_intelligence" title="Artificial intelligence">artificial intelligence</a>, <a href="/wiki/Philosophy_of_mind" title="Philosophy of mind">philosophy</a>, <a href="/wiki/Neuroscience" title="Neuroscience">neuroscience</a>, and <a href="/wiki/Anthropology" title="Anthropology">anthropology</a>.<sup id="cite_ref-stanford1_49-0" class="reference"><a href="#cite_note-stanford1-49">&#91;48&#93;</a></sup></li>
<li><b><a href="/wiki/Comma-separated_values" title="Comma-separated values">Comma-separated values</a></b> –  (<b>CSV</b>) <a href="/wiki/Computer_file" title="Computer file">file</a> is a delimited <a href="/wiki/Text_file" title="Text file">text file</a> that uses a comma to separate values. A CSV file stores <a href="/wiki/Table_(information)" title="Table (information)">tabular</a> data (numbers and text) in <a href="/wiki/Plain_text" title="Plain text">plain text</a>.  Each line of the file is a data <a href="/wiki/Record_(computer_science)" title="Record (computer science)">record</a>.  Each record consists of one or more <a href="/wiki/Field_(computer_science)" title="Field (computer science)">fields</a>, separated by <a href="/wiki/Comma" title="Comma">commas</a>. The use of the comma as a field separator is the source of the name for this <a href="/wiki/File_format" title="File format">file format</a>.</li>
<li><b><a href="/wiki/Compiler" title="Compiler">Compiler</a></b> – is a <a href="/wiki/Computer_program" title="Computer program">computer program</a> that transforms computer code written in one <a href="/wiki/Programming_language" title="Programming language">programming language</a> (the source language) into another programming language (the target language). Compilers are a type of <a href="/wiki/Translator_(computing)" title="Translator (computing)">translator</a> that support digital devices, primarily computers. The name <i>compiler</i> is primarily used for programs that translate <a href="/wiki/Source_code" title="Source code">source code</a> from a <a href="/wiki/High-level_programming_language" title="High-level programming language">high-level programming language</a> to a <a href="/wiki/Lower_level_language" class="mw-redirect" title="Lower level language">lower level language</a> (e.g., <a href="/wiki/Assembly_language" title="Assembly language">assembly language</a>, <a href="/wiki/Object_code" title="Object code">object code</a>, or <a href="/wiki/Machine_code" title="Machine code">machine code</a>) to create an <a href="/wiki/Executable" title="Executable">executable</a> program.<sup id="cite_ref-50" class="reference"><a href="#cite_note-50">&#91;49&#93;</a></sup></li>
<li><b><a href="/wiki/Compiler_theory" class="mw-redirect" title="Compiler theory">Compiler theory</a></b> –</li>
<li><b><a href="/wiki/Computability_theory" title="Computability theory">Computability theory</a></b> – also known as <b>recursion theory</b>, is a branch of <a href="/wiki/Mathematical_logic" title="Mathematical logic">mathematical logic</a>, of <a href="/wiki/Computer_science" title="Computer science">computer science</a>, and of the <a href="/wiki/Theory_of_computation" title="Theory of computation">theory of computation</a> that originated in the 1930s with the study of <a href="/wiki/Computable_function" title="Computable function">computable functions</a> and <a href="/wiki/Turing_degree" title="Turing degree">Turing degrees</a>. The field has since expanded to include the study of generalized computability and definability. In these areas, recursion theory overlaps with <a href="/wiki/Proof_theory" title="Proof theory">proof theory</a> and <a href="/wiki/Effective_descriptive_set_theory" title="Effective descriptive set theory">effective descriptive set theory</a>.</li>
<li><b><a href="/wiki/Computation" title="Computation">Computation</a></b> – is any type of <a href="/wiki/Calculation" title="Calculation">calculation</a><sup id="cite_ref-51" class="reference"><a href="#cite_note-51">&#91;50&#93;</a></sup><sup id="cite_ref-52" class="reference"><a href="#cite_note-52">&#91;51&#93;</a></sup> that includes both arithmetical and non-arithmetical steps and follows a well-defined <a href="/wiki/Model_(abstract)" class="mw-redirect" title="Model (abstract)">model</a>, for example an <a href="/wiki/Algorithm" title="Algorithm">algorithm</a>. The study of computation is paramount to the discipline of <a href="/wiki/Computer_science" title="Computer science">computer science</a>.</li>
<li><b><a href="/wiki/Computational_biology" title="Computational biology">Computational biology</a></b> –</li>
<li><b><a href="/wiki/Computational_chemistry" title="Computational chemistry">Computational chemistry</a></b> – is a branch of <a href="/wiki/Chemistry" title="Chemistry">chemistry</a> that uses <a href="/wiki/Computer_simulation" title="Computer simulation">computer simulation</a> to assist in solving chemical problems. It uses methods of <a href="/wiki/Theoretical_chemistry" title="Theoretical chemistry">theoretical chemistry</a>, incorporated into efficient <a href="/wiki/Computer_program" title="Computer program">computer programs</a>, to calculate the structures and properties of <a href="/wiki/Molecule" title="Molecule">molecules</a> and solids.</li>

<li><b><a href="/wiki/Computer_programming" title="Computer programming">Computer programming</a></b> – is the process of designing and building an <a href="/wiki/Executable" title="Executable">executable</a> <a href="/wiki/Computer_program" title="Computer program">computer program</a> for accomplishing a specific <a href="/wiki/Computing" title="Computing">computing</a> task. Programming involves tasks such as analysis, generating <a href="/wiki/Algorithm" title="Algorithm">algorithms</a>, profiling algorithms' accuracy and resource consumption, and the implementation of algorithms in a chosen <a href="/wiki/Programming_language" title="Programming language">programming language</a> (commonly referred to as <b>coding</b><sup id="cite_ref-tumblr2014_46-1" class="reference"><a href="#cite_note-tumblr2014-46">&#91;45&#93;</a></sup><sup id="cite_ref-tumblr1_47-1" class="reference"><a href="#cite_note-tumblr1-47">&#91;46&#93;</a></sup>). The <a href="/wiki/Source_code" title="Source code">source code</a> of a program is written in one or more programming languages. The purpose of programming is to find a sequence of instructions that will automate the performance of a task for solving a given problem. The process of programming thus often requires expertise in several different subjects, including knowledge of the <a href="/wiki/Domain_(software_engineering)" title="Domain (software engineering)">application domain</a>, specialized algorithms, and formal <a href="/wiki/Logic" title="Logic">logic</a>.</li>

<li><b><a href="/wiki/Concurrency_(computer_science)" title="Concurrency (computer science)">Concurrency</a></b> – refers to the ability of different parts or units of a program, algorithm, or problem to be executed out-of-order or in partial order, without affecting the final outcome.  This allows for parallel execution of the concurrent units, which can significantly improve overall speed of the execution in multi-processor and multi-core systems. In more technical terms, concurrency refers to the decomposability property of a program, algorithm, or problem into order-independent or partially-ordered components or units.<sup id="cite_ref-68" class="reference"><a href="#cite_note-68">&#91;67&#93;</a></sup></li>

<li><b><a href="/wiki/Declaration_(computer_programming)" title="Declaration (computer programming)">Declaration</a></b> – In <a href="/wiki/Computer_programming" title="Computer programming">computer programming</a>, a <b>declaration</b> is a <a href="/wiki/Language_construct" title="Language construct">language construct</a> that specifies properties of an <a href="/wiki/Identifier_(computer_programming)" class="mw-redirect" title="Identifier (computer programming)">identifier</a>: it declares what a word (identifier) "means".<sup id="cite_ref-c11_83-0" class="reference"><a href="#cite_note-c11-83">&#91;82&#93;</a></sup> Declarations are most commonly used for <a href="/wiki/Subroutine" title="Subroutine">functions</a>, <a href="/wiki/Variable_(computer_science)" title="Variable (computer science)">variables</a>, <a href="/wiki/Constant_(computer_programming)" title="Constant (computer programming)">constants</a>, and <a href="/wiki/Class_(computer_programming)" title="Class (computer programming)">classes</a>, but can also be used for other entities such as enumerations and type definitions.<sup id="cite_ref-c11_83-1" class="reference"><a href="#cite_note-c11-83">&#91;82&#93;</a></sup> Beyond the name (the identifier itself) and the kind of entity (function, variable, etc.), declarations typically specify the <a href="/wiki/Data_type" title="Data type">data type</a> (for variables and constants), or the <a href="/wiki/Type_signature" title="Type signature">type signature</a> (for functions); types may also include dimensions, such as for arrays. A declaration is used to announce the existence of the entity to the <a href="/wiki/Compiler" title="Compiler">compiler</a>; this is important in those <a href="/wiki/Strongly_typed" class="mw-redirect" title="Strongly typed">strongly typed</a> languages that require functions, variables, and constants, and their types to be specified with a declaration before use, and is used in <a href="/wiki/Forward_declaration" title="Forward declaration">forward declaration</a>.<sup id="cite_ref-84" class="reference"><a href="#cite_note-84">&#91;83&#93;</a></sup> The term "declaration" is frequently contrasted with the term "definition",<sup id="cite_ref-c11_83-2" class="reference"><a href="#cite_note-c11-83">&#91;82&#93;</a></sup> but meaning and usage varies significantly between languages.</li>

<li><b><a href="/wiki/Distributed_computing" title="Distributed computing">Distributed computing</a></b> – is a field of <a href="/wiki/Computer_science" title="Computer science">computer science</a> that studies distributed systems. A <i>distributed system</i> is a system whose components are located on different <a href="/wiki/Computer_network" title="Computer network">networked computers</a>, which communicate and coordinate their actions by <a href="/wiki/Message_passing" title="Message passing">passing messages</a> to one another.<sup id="cite_ref-Coulouris_86-0" class="reference"><a href="#cite_note-Coulouris-86">&#91;85&#93;</a></sup> The components interact with one another in order to achieve a common goal. Three significant characteristics of distributed systems are: concurrency of components, <a href="/wiki/Clock_synchronization" title="Clock synchronization">lack of a global clock</a>, and independent failure of components.<sup id="cite_ref-Coulouris_86-1" class="reference"><a href="#cite_note-Coulouris-86">&#91;85&#93;</a></sup> Examples of distributed systems vary from <a href="/wiki/Service-oriented_architecture" title="Service-oriented architecture">SOA-based systems</a> to <a href="/wiki/Massively_multiplayer_online_game" title="Massively multiplayer online game">massively multiplayer online games</a> to <a href="/wiki/Peer-to-peer" title="Peer-to-peer">peer-to-peer applications</a>.</li>
<li><b><a href="/wiki/Divide_and_conquer_algorithm" class="mw-redirect" title="Divide and conquer algorithm">Divide and conquer algorithm</a></b> – is an <a href="/wiki/Algorithm_design_paradigm" class="mw-redirect" title="Algorithm design paradigm">algorithm design paradigm</a> based on multi-branched <a href="/wiki/Recursion" title="Recursion">recursion</a>. A divide-and-conquer <a href="/wiki/Algorithm" title="Algorithm">algorithm</a> works by recursively breaking down a problem into two or more sub-problems of the same or related type, until these become simple enough to be solved directly. The solutions to the sub-problems are then combined to give a solution to the original problem.</li>

<li><b><a href="/wiki/Software_documentation" title="Software documentation">Documentation</a></b> – <b>Software documentation</b> is written text or illustration that accompanies <a href="/wiki/Computer_software" class="mw-redirect" title="Computer software">computer software</a> or is embedded in the source code.  It either explains how it operates or how to use it, and may mean different things to people in different roles.</li>
<li><b><a href="/wiki/Domain_Name_System" title="Domain Name System">Domain Name System</a></b> –  (<b>DNS</b>) is a <a href="/wiki/Hierarchy" title="Hierarchy">hierarchical</a> and <a href="/wiki/Decentralised_system" title="Decentralised system">decentralized</a> naming system for computers, services, or other resources connected to the <a href="/wiki/Internet" title="Internet">Internet</a> or a private network. It associates various information with <a href="/wiki/Domain_name" title="Domain name">domain names</a> assigned to each of the participating entities. Most prominently, it translates more readily memorized domain names to the numerical <a href="/wiki/IP_address" title="IP address">IP addresses</a> needed for locating and identifying computer services and devices with the underlying <a href="/wiki/Communication_protocol" title="Communication protocol">network protocols</a>. By providing a worldwide, <a href="/wiki/Distributed_computing" title="Distributed computing">distributed</a> <a href="/wiki/Directory_service" title="Directory service">directory service</a>, the Domain Name System has been an essential component of the functionality of the Internet since 1985.</li>
<li><b><a href="/wiki/Double-precision_floating-point_format" title="Double-precision floating-point format">Double-precision floating-point format</a></b> –  is a <a href="/wiki/Computer_number_format" title="Computer number format">computer number format</a>. It represents a wide <a href="/wiki/Dynamic_range" title="Dynamic range">dynamic range</a> of numeric values by using a floating radix point.</li>
<li><b><a href="/wiki/Download" title="Download">Download</a></b> – In <a href="/wiki/Computer_network" title="Computer network">computer networks</a>, <b>download</b> means to receive data from a remote system, typically a <a href="/wiki/Server_(computing)" title="Server (computing)">server</a><sup id="cite_ref-main_87-0" class="reference"><a href="#cite_note-main-87">&#91;86&#93;</a></sup> such as a <a href="/wiki/Web_server" title="Web server">web server</a>, an <a href="/wiki/File_Transfer_Protocol" title="File Transfer Protocol">FTP</a> server, an <a href="/wiki/Email" title="Email">email</a> server, or other similar systems. This contrasts with <a href="/wiki/Upload" title="Upload">uploading</a>, where data is sent <i>to</i> a remote server. A <i>download</i> is a <a href="/wiki/Computer_file" title="Computer file">file</a> offered for downloading or that has been downloaded, or the process of receiving such a file.</li>
<li><b><a href="/wiki/Edge_device" title="Edge device">Edge device</a></b> – is a device which provides an entry point into enterprise or service provider core networks. Examples include <a href="/wiki/Router_(computing)" title="Router (computing)">routers</a>, routing <a href="/wiki/Network_switch" title="Network switch">switches</a>, <a href="/wiki/Integrated_access_device" title="Integrated access device">integrated access devices</a> (IADs), multiplexers, and a variety of <a href="/wiki/Metropolitan_area_network" title="Metropolitan area network">metropolitan area network</a> (MAN) and <a href="/wiki/Wide_area_network" title="Wide area network">wide area network</a> (WAN) access devices.  Edge devices also provide connections into carrier and service provider networks. An edge device that connects a <a href="/wiki/Local_area_network" title="Local area network">local area network</a> to a high speed switch or backbone (such as an ATM switch) may be called an edge concentrator.</li>

<li><b><a href="/wiki/Event_(computing)" title="Event (computing)">Event</a></b> –  Is an action or occurrence recognized by software, often originating <a href="/wiki/Asynchronous_I/O" title="Asynchronous I/O">asynchronously</a> from the external environment, that may be <a href="/wiki/Event_handler" class="mw-redirect" title="Event handler">handled</a> by the software. <li><b><a href="/wiki/Event-driven_programming" title="Event-driven programming">Event-driven programming</a></b> – is a <a href="/wiki/Programming_paradigm" title="Programming paradigm">programming paradigm</a> in which the <a href="/wiki/Control_flow" title="Control flow">flow of the program</a> is determined by <a href="/wiki/Event_(computing)" title="Event (computing)">events</a> such as user actions (<a href="/wiki/Computer_mouse" title="Computer mouse">mouse</a> clicks, key presses), <a href="/wiki/Sensor" title="Sensor">sensor</a> outputs, or <a href="/wiki/Message_passing" title="Message passing">messages</a> from other programs or <a href="/wiki/Thread_(computer_science)" class="mw-redirect" title="Thread (computer science)">threads</a>. Event-driven programming is the dominant paradigm used in <a href="/wiki/Graphical_user_interface" title="Graphical user interface">graphical user interfaces</a> and other applications (e.g., JavaScript <a href="/wiki/Web_application" title="Web application">web applications</a>) that are centered on performing certain actions in response to <a href="/wiki/Input/output" title="Input/output">user input</a>. This is also true of programming for <a href="/wiki/Device_driver" title="Device driver">device drivers</a> (e.g., <a href="/wiki/P_(programming_language)" title="P (programming language)">P</a> in USB device driver stacks<sup id="cite_ref-88" class="reference"><a href="#cite_note-88">&#91;87&#93;</a></sup>).</li>
<li><b><a href="/wiki/Evolutionary_computing" class="mw-redirect" title="Evolutionary computing">Evolutionary computing</a></b> – is a family of <a href="/wiki/Algorithm" title="Algorithm">algorithms</a> for <a href="/wiki/Global_optimization" title="Global optimization">global optimization</a> inspired by <a href="/wiki/Biological_evolution" class="mw-redirect" title="Biological evolution">biological evolution</a>, and the subfield of <a href="/wiki/Artificial_intelligence" title="Artificial intelligence">artificial intelligence</a> and <a href="/w/index.php?title=Soft_computing&amp;action=edit&amp;redlink=1" class="new" title="Soft computing (page does not exist)">soft computing</a> studying these algorithms. In technical terms, they are a family of population-based <a href="/wiki/Trial_and_error" title="Trial and error">trial and error</a> problem solvers with a <a href="/wiki/Metaheuristic" title="Metaheuristic">metaheuristic</a> or <a href="/wiki/Stochastic_optimization" title="Stochastic optimization">stochastic optimization</a> character.</li>
<li><b><a href="/wiki/Executable" title="Executable">Executable</a></b> – <b>executable code</b> or an <b>executable file</b> or <b>executable program</b>, sometimes simply referred to as an  <b>executable</b>, causes a computer "to perform indicated tasks according to encoded <a href="/wiki/Instruction_(computer_science)" class="mw-redirect" title="Instruction (computer science)">instructions</a>,"<sup id="cite_ref-89" class="reference"><a href="#cite_note-89">&#91;88&#93;</a></sup> as opposed to a <a href="/wiki/Data_(computing)" title="Data (computing)">data file</a> that must be <a href="/wiki/Parser" class="mw-redirect" title="Parser">parsed</a> by a program to be meaningful. The exact interpretation depends upon the use - while "instructions" is traditionally taken to mean <a href="/wiki/Machine_code" title="Machine code">machine code</a> instructions for a physical <a href="/wiki/Central_processing_unit" title="Central processing unit">CPU</a>, in some contexts a file containing <a href="/wiki/Bytecode" title="Bytecode">bytecode</a> or <a href="/wiki/Scripting_language" title="Scripting language">scripting language</a> instructions may also be considered executable.</li>
<li><b><a href="/wiki/Exception_handling" title="Exception handling">Exception handling</a></b> – is the process of responding to the occurrence, during <a href="/wiki/Computation" title="Computation">computation</a>, of <i>exceptions</i> – anomalous or exceptional conditions requiring special processing – often disrupting the normal flow of <a href="/wiki/Computer_program" title="Computer program">program</a> <a href="/wiki/Execution_(computing)" title="Execution (computing)">execution</a>. <li><b><a href="/wiki/Game_theory" title="Game theory">Game theory</a></b> – is the study of <a href="/wiki/Mathematical_model" title="Mathematical model">mathematical models</a> of strategic interaction between rational decision-makers.<sup id="cite_ref-Myerson_97-0" class="reference"><a href="#cite_note-Myerson-97">&#91;96&#93;</a></sup> It has applications in all fields of <a href="/wiki/Social_science" title="Social science">social science</a>, as well as in <a href="/wiki/Logic" title="Logic">logic</a> and <a href="/wiki/Computer_science" title="Computer science">computer science</a>. Originally, it addressed <a href="/wiki/Zero-sum_game" title="Zero-sum game">zero-sum games</a>, in which each participant's gains or losses are exactly balanced by those of the other participants. Today, game theory applies to a wide range of behavioral relations, and is now an <a href="/wiki/Umbrella_term" class="mw-redirect" title="Umbrella term">umbrella term</a> for the <a href="/wiki/Science" title="Science">science</a> of logical decision making in humans, animals, and computers.</li>

<li><b><a href="/wiki/Gigabyte" title="Gigabyte">Gigabyte</a></b> –  is a multiple of the unit <a href="/wiki/Byte" title="Byte">byte</a> for digital information. The <a href="/wiki/SI_prefix" class="mw-redirect" title="SI prefix">prefix</a> <i><a href="/wiki/Giga-" title="Giga-">giga</a></i> means 10<sup>9</sup> in the <a href="/wiki/International_System_of_Units" title="International System of Units">International System of Units</a> (SI). Therefore, one gigabyte is <span style="white-space:nowrap">1<span style="margin-left:0.25em">000</span><span style="margin-left:0.25em">000</span><span style="margin-left:0.25em">000</span><span style="margin-left:0.25em">bytes</span></span>. <li><b><a href="/wiki/Global_variable" title="Global variable">Global variable</a></b> – In <a href="/wiki/Computer_programming" title="Computer programming">computer programming</a>, a global variable is a variable with global <a href="/wiki/Scope_(programming)" class="mw-redirect" title="Scope (programming)">scope</a>, meaning that it is visible (hence accessible) throughout the program, unless <a href="/wiki/Variable_shadowing" title="Variable shadowing">shadowed</a>. The set of all global variables is known as the <i>global environment</i> or <i>global state.</i> In compiled languages, global variables are generally <a href="/wiki/Static_variable" title="Static variable">static variables</a>, whose <a href="/wiki/Variable_(programming)#Scope_and_extent" class="mw-redirect" title="Variable (programming)">extent</a> (lifetime) is the entire runtime of the program, though in interpreted languages (including <a href="/wiki/Command-line_interpreter" class="mw-redirect" title="Command-line interpreter">command-line interpreters</a>), global variables are generally dynamically allocated when declared, since they are not known ahead of time.</li>
<li><b><a href="/wiki/Graph_theory" title="Graph theory">Graph theory</a></b> – In <a href="/wiki/Mathematics" title="Mathematics">mathematics</a>, graph theory is the study of <i><a href="/wiki/Graph_(discrete_mathematics)" title="Graph (discrete mathematics)">graphs</a></i>, which are mathematical structures used to model pairwise relations between objects. A graph in this context is made up of <i><a href="/wiki/Vertex_(graph_theory)" title="Vertex (graph theory)">vertices</a></i> (also called <i>nodes</i> or <i>points</i>) which are connected by <i><a href="/wiki/Glossary_of_graph_theory_terms#edge" title="Glossary of graph theory terms">edges</a></i> (also called <i>links</i> or <i>lines</i>). A distinction is made between undirected graphs, where edges link two vertices symmetrically, and directed graphs, where edges link two vertices asymmetrically.</li>

<li><b><a href="/wiki/Hash_function" title="Hash function">Hash function</a></b> – is any <a href="/wiki/Function_(mathematics)" title="Function (mathematics)">function</a> that can be used to map <a href="/wiki/Data_(computing)" title="Data (computing)">data</a> of arbitrary size to data of a fixed size. The values returned by a hash function are called <b>hash values</b>, <b>hash codes</b>, <b>digests</b>, or simply <b>hashes</b>. Hash functions are often used in combination with a <a href="/wiki/Hash_table" title="Hash table">hash table</a>, a common <a href="/wiki/Data_structure" title="Data structure">data structure</a> used in computer software for rapid data lookup. Hash functions accelerate table or database lookup by detecting duplicated records in a large file.</li>
<li><b><a href="/wiki/Hash_table" title="Hash table">Hash table</a></b> – In <a href="/wiki/Computing" title="Computing">computing</a>, a <b>hash table</b> (<b>hash map</b>) is a <a href="/wiki/Data_structure" title="Data structure">data structure</a> that implements an <a href="/wiki/Associative_array" title="Associative array">associative array</a> <a href="/wiki/Abstract_data_type" title="Abstract data type">abstract data type</a>, a structure that can map <a href="/wiki/Unique_key" title="Unique key">keys</a> to <a href="/wiki/Value_(computer_science)" title="Value (computer science)">values</a>. A hash table uses a <a href="/wiki/Hash_function" title="Hash function">hash function</a> to compute an <i>index</i> into an array of <i>buckets</i> or <i>slots</i>, from which the desired value can be found.</li>
<li><b><a href="/wiki/Heap_(data_structure)" title="Heap (data structure)">Heap</a></b> – is a specialized <a href="/wiki/Tree_(data_structure)" title="Tree (data structure)">tree</a>-based <a href="/wiki/Data_structure" title="Data structure">data structure</a> which is essentially an almost complete<sup id="cite_ref-98" class="reference"><a href="#cite_note-98">&#91;97&#93;</a></sup> tree that satisfies the <i>heap property:</i> if P is a parent <a href="/wiki/Node_(computer_science)" title="Node (computer science)">node</a> of C, then the <i>key</i> (the <i>value</i>) of P is either greater than or equal to (in a <i>max heap</i>) or less than or equal to (in a <i>min heap</i>) the key of C.<sup id="cite_ref-99" class="reference"><a href="#cite_note-99">&#91;98&#93;</a></sup> The node at the "top" of the heap (with no parents) is called the <i>root</i> node.</li>
<li><b><a href="/wiki/Heapsort" title="Heapsort">Heapsort</a></b> – is a <a href="/wiki/Comparison_sort" title="Comparison sort">comparison-based</a> <a href="/wiki/Sorting_algorithm" title="Sorting algorithm">sorting algorithm</a>. Heapsort can be thought of as an improved <a href="/wiki/Selection_sort" title="Selection sort">selection sort</a>: like that algorithm, it divides its input into a sorted and an unsorted region, and it iteratively shrinks the unsorted region by extracting the largest element and moving that to the sorted region. The improvement consists of the use of a <a href="/wiki/Heap_(data_structure)" title="Heap (data structure)">heap</a> data structure rather than a linear-time search to find the maximum.<sup id="cite_ref-100" class="reference"><a href="#cite_note-100">&#91;99&#93;</a></sup></li>

<li><b><a href="/wiki/Imperative_programming" title="Imperative programming">Imperative programming</a></b> – is a  <a href="/wiki/Programming_paradigm" title="Programming paradigm">programming paradigm</a> that uses <a href="/wiki/Statement_(computer_science)" title="Statement (computer science)">statements</a> that change a program's <a href="/wiki/State_(computer_science)" title="State (computer science)">state</a>. In much the same way that the <a href="/wiki/Imperative_mood" title="Imperative mood">imperative mood</a> in <a href="/wiki/Natural_language" title="Natural language">natural languages</a> expresses commands, an imperative program consists of <a href="/wiki/Command_(computing)" title="Command (computing)">commands</a> for the computer to perform. Imperative programming focuses on describing <i>how</i> a program operates.</li>
<li><b><a href="/wiki/Incremental_build_model" title="Incremental build model">Incremental build model</a></b> – is a method of <a href="/wiki/Software_development" title="Software development">software development</a> where the product is <a href="/wiki/Software_design" title="Software design">designed</a>, implemented and <a href="/wiki/Software_testing" title="Software testing">tested</a> incrementally (a little more is added each time) until the product is finished. It involves both development and maintenance. The product is defined as finished when it satisfies all of its requirements. This model combines the elements of the <a href="/wiki/Waterfall_model" title="Waterfall model">waterfall model</a> with the iterative philosophy of <a href="/wiki/Software_prototyping" title="Software prototyping">prototyping</a>.</li>
<li><b><a href="/wiki/Information_space_analysis" title="Information space analysis">Information space analysis</a></b> – is a <a href="/wiki/Deterministic" class="mw-redirect" title="Deterministic">deterministic</a> method, enhanced by <a href="/wiki/Artificial_intelligence" title="Artificial intelligence">machine intelligence</a>, for locating and assessing <a href="/wiki/Resource" title="Resource">resources</a> for team-centric efforts.</li>

<li><b><a href="/wiki/Inheritance_(computer_science)" class="mw-redirect" title="Inheritance (computer science)">Inheritance</a></b> – In <a href="/wiki/Object-oriented_programming" title="Object-oriented programming">object-oriented programming</a>, inheritance is the mechanism of basing an <a href="/wiki/Object_(computer_science)" title="Object (computer science)">object</a> or <a href="/wiki/Class_(computer_programming)" title="Class (computer programming)">class</a> upon another object (<a href="/wiki/Prototype-based_programming" title="Prototype-based programming">prototype-based inheritance</a>) or class (<a href="/wiki/Class-based_programming" title="Class-based programming">class-based inheritance</a>), retaining similar implementation. Also defined as deriving new classes (<a href="#Subclasses_and_superclasses">sub classes</a>) from existing ones (super class or <a href="/wiki/Fragile_base_class" title="Fragile base class">base class</a>) and forming them into a hierarchy of classes.</li>
<li><b><a href="/wiki/Input/output" title="Input/output">Input/output</a></b> – or I/O (or, informally, io or IO), is the communication between an <a href="/wiki/Information_processing_system" class="mw-redirect" title="Information processing system">information processing system</a>, such as a <a href="/wiki/Computer" title="Computer">computer</a>, and the outside world, possibly a human or another <a href="/wiki/Information_processor" title="Information processor">information processing system</a>. <a href="/wiki/Information" title="Information">Inputs</a> are the signals or data received by the system and outputs are the signals or <a href="/wiki/Data_(computing)" title="Data (computing)">data</a> sent from it. The term can also be used as part of an action; to "perform I/O" is to perform an <a href="/wiki/I/O_scheduling" title="I/O scheduling">input or output operation</a>.</li>
<li><b><a href="/wiki/Insertion_sort" title="Insertion sort">Insertion sort</a></b> – is a simple <a href="/wiki/Sorting_algorithm" title="Sorting algorithm">sorting algorithm</a> that builds the final <a href="/wiki/Sorted_array" title="Sorted array">sorted array</a> (or list) one item at a time.</li>
<li><b><a href="/wiki/Instruction_cycle" title="Instruction cycle">Instruction cycle</a></b> – (also known as the fetch–decode–execute cycle or simply the fetch-execute cycle), is the cycle which the <a href="/wiki/Central_processing_unit" title="Central processing unit">central processing unit</a> (CPU) follows from <a href="/wiki/Booting" title="Booting">boot-up</a> until the computer has shut down in order to process instructions. It is composed of three main stages: the fetch stage, the decode stage, and the execute stage.</li>
<li><b><a href="/wiki/Integer_(computer_science)" title="Integer (computer science)">Integer</a></b> – is a <a href="/wiki/Data" title="Data">datum</a> of integral data type, a <a href="/wiki/Data_type" title="Data type">data type</a> that represents some <a href="/wiki/Interval_(mathematics)" title="Interval (mathematics)">range</a> of mathematical <a href="/wiki/Integer" title="Integer">integers</a>. Integral data types may be of different sizes and may or may not be allowed to contain negative values. Integers are commonly represented in a computer as a group of binary digits (bits). The size of the grouping varies so the set of integer sizes available varies between different types of computers. Computer hardware, including <a href="/wiki/Virtual_machine" title="Virtual machine">virtual machines</a>, nearly always provide a way to represent a processor <a href="/wiki/Word_size" class="mw-redirect" title="Word size">register</a> or memory address as an integer.</li>
<li><b><a href="/wiki/Integrated_development_environment" title="Integrated development environment">Integrated development environment</a></b> – (IDE), is a <a href="/wiki/Application_software" title="Application software">software application</a> that provides comprehensive facilities to <a href="/wiki/Computer_programmer" class="mw-redirect" title="Computer programmer">computer programmers</a> for <a href="/wiki/Software_development" title="Software development">software development</a>. An IDE normally consists of at least a <a href="/wiki/Source_code_editor" class="mw-redirect" title="Source code editor">source code editor</a>, <a href="/wiki/Build_automation" title="Build automation">build automation</a> tools, and a <a href="/wiki/Debugger" title="Debugger">debugger</a>.</li>
<li><b><a href="/wiki/Integration_testing" title="Integration testing">Integration testing</a></b> – (sometimes called integration and testing, abbreviated I&amp;T) is the phase in <a href="/wiki/Software_testing" title="Software testing">software testing</a> in which individual software modules are combined and tested as a group. Integration testing is conducted to evaluate the <a href="/wiki/Regulatory_compliance" title="Regulatory compliance">compliance</a> of a system or component with specified <a href="/wiki/Functional_requirement" title="Functional requirement">functional requirements</a>.<sup id="cite_ref-101" class="reference"><a href="#cite_note-101">&#91;100&#93;</a></sup> It occurs after <a href="/wiki/Unit_testing" title="Unit testing">unit testing</a> and before <a href="/wiki/Software_verification_and_validation" title="Software verification and validation">validation testing</a>. Integration testing takes as its input <a href="/wiki/Module_(programming)" class="mw-redirect" title="Module (programming)">modules</a> that have been unit tested, groups them in larger aggregates, applies tests defined in an integration <a href="/wiki/Test_plan" title="Test plan">test plan</a> to those aggregates, and delivers as its output the integrated system ready for <a href="/wiki/System_testing" title="System testing">system testing</a>.<sup id="cite_ref-102" class="reference"><a href="#cite_note-102">&#91;101&#93;</a></sup></li>
<li><b><a href="/wiki/Intellectual_property" title="Intellectual property">Intellectual property</a></b> – (IP), is a category of <a href="/wiki/Property" title="Property">property</a> that includes intangible creations of the human intellect.<sup id="cite_ref-103" class="reference"><a href="#cite_note-103">&#91;102&#93;</a></sup><sup id="cite_ref-104" class="reference"><a href="#cite_note-104">&#91;103&#93;</a></sup> There are many types of intellectual property, and some countries recognize more than others.<sup id="cite_ref-105" class="reference"><a href="#cite_note-105">&#91;104&#93;</a></sup><sup id="cite_ref-106" class="reference"><a href="#cite_note-106">&#91;105&#93;</a></sup><sup id="cite_ref-107" class="reference"><a href="#cite_note-107">&#91;106&#93;</a></sup><sup id="cite_ref-108" class="reference"><a href="#cite_note-108">&#91;107&#93;</a></sup><sup id="cite_ref-109" class="reference"><a href="#cite_note-109">&#91;108&#93;</a></sup> The most well-known types are <a href="/wiki/Copyright" title="Copyright">copyrights</a>, <a href="/wiki/Patent" title="Patent">patents</a>, <a href="/wiki/Trademark" title="Trademark">trademarks</a>, and <a href="/wiki/Trade_secret" title="Trade secret">trade secrets</a>.</li>

<li><b><a href="/wiki/Interface_(computing)" title="Interface (computing)">Interface</a></b> – is a shared boundary across which two or more separate components of a <a href="/wiki/Computer_system" class="mw-redirect" title="Computer system">computer system</a> exchange information. The exchange can be between <a href="/wiki/Software" title="Software">software</a>, <a href="/wiki/Computer_hardware" title="Computer hardware">computer hardware</a>, <a href="/wiki/Peripheral" title="Peripheral">peripheral</a> devices, <a href="/wiki/User_interface" title="User interface">humans</a>, and combinations of these.<sup id="cite_ref-HookwayInterface14_110-0" class="reference"><a href="#cite_note-HookwayInterface14-110">&#91;109&#93;</a></sup> Some computer hardware devices, such as a <a href="/wiki/Touchscreen" title="Touchscreen">touchscreen</a>, can both send and receive data through the interface, while others such as a mouse or microphone may only provide an interface to send data to a given system.<sup id="cite_ref-111" class="reference"><a href="#cite_note-111">&#91;110&#93;</a></sup></li>
<li><b><a href="/wiki/Internal_documentation" title="Internal documentation">Internal documentation</a></b> – Computer <a href="/wiki/Software" title="Software">software</a> is said to have Internal Documentation if the notes on how and why various parts of code operate is included within the <a href="/wiki/Source_code" title="Source code">source code</a> as comments.  It is often combined with meaningful <a href="/wiki/Variable_(programming)" class="mw-redirect" title="Variable (programming)">variable</a> names with the intention of providing potential future programmers a means of understanding the workings of the code. This contrasts with external <a href="/wiki/Documentation" title="Documentation">documentation</a>, where programmers keep their notes and explanations in a separate document.</li>
<li><b><a href="/wiki/Internet" title="Internet">Internet</a></b> – The Internet (<a href="/wiki/Portmanteau" title="Portmanteau">portmanteau</a> of interconnected network) is the global system of interconnected <a href="/wiki/Computer_network" title="Computer network">computer networks</a> that use the <a href="/wiki/Internet_protocol_suite" title="Internet protocol suite">Internet protocol suite</a> (TCP/IP) to link devices worldwide. It is a <i>network of networks</i> that consists of private, public, academic, business, and government networks of local to global scope, linked by a broad array of electronic, wireless, and optical networking technologies.</li>

<li><b><a href="/wiki/Kernel_(operating_system)" title="Kernel (operating system)">Kernel</a></b> – When referring to an <a href="/wiki/Operating_system" title="Operating system">operating system</a>, the kernel is the first section of the <a href="/wiki/Operating_system" title="Operating system">operating system</a> to load into <a href="/wiki/Memory" title="Memory">memory</a>. As the center of the <a href="/wiki/Operating_system" title="Operating system">operating system</a>, the kernel needs to be small, efficient and loaded into a protected area in the memory so as not to be overwritten. It can be responsible for such things as disk drive management, file management, memory management, process management, etc.</li>
<li><b><a href="/wiki/Library_(computing)" title="Library (computing)">Library (computing)</a></b> – is a collection of <a href="/wiki/Non-volatile_memory" title="Non-volatile memory">non-volatile</a> resources used by <a href="/wiki/Computer_program" title="Computer program">computer programs</a>, often for <a href="/wiki/Software_development" title="Software development">software development</a>. <li><b><a href="/wiki/Linear_search" title="Linear search">Linear search</a></b> – or sequential search, is a method for finding an element within a <a href="/wiki/List_(computing)" class="mw-redirect" title="List (computing)">list</a>. It sequentially checks each element of the list until a match is found or the whole list has been searched.<sup id="cite_ref-FOOTNOTEKnuth1998§6.1_(&quot;Sequential_search&quot;)_117-0" class="reference"><a href="#cite_note-FOOTNOTEKnuth1998§6.1_(&quot;Sequential_search&quot;)-117">&#91;116&#93;</a></sup></li>
<li><b><a href="/wiki/Linked_list" title="Linked list">Linked list</a></b> – is a linear collection of data elements, whose order is not given by their physical placement in memory. Instead, each element <a href="/wiki/Pointer_(computer_programming)" title="Pointer (computer programming)">points</a> to the next. It is a <a href="/wiki/Data_structure" title="Data structure">data structure</a> consisting of a collection of <a href="/wiki/Node_(computer_science)" title="Node (computer science)">nodes</a> which together represent a <a href="/wiki/Sequence" title="Sequence">sequence</a>.</li>
<li><b><a href="/wiki/Linker_(computing)" title="Linker (computing)">Linker</a></b> –  or link editor, is a computer <a href="/wiki/System_utility" class="mw-redirect" title="System utility">utility</a> program that takes one or more <a href="/wiki/Object_file" title="Object file">object files</a> generated by a <a href="/wiki/Compiler" title="Compiler">compiler</a> or an <a href="/wiki/Assembler_(computing)" class="mw-redirect" title="Assembler (computing)">assembler</a> and combines them into a single <a href="/wiki/Executable" title="Executable">executable</a> file, <a href="/wiki/Library_(computing)" title="Library (computing)">library</a> file, or another 'object' file.  A simpler version that writes its output directly to memory is called the <i>loader</i>, though <a href="/wiki/Loader_(computing)" title="Loader (computing)">loading</a> is typically considered a separate process.<sup id="cite_ref-118" class="reference"><a href="#cite_note-118">&#91;117&#93;</a></sup></li>
<li><b><a href="/wiki/List_(abstract_data_type)" title="List (abstract data type)">List</a></b> – or sequence, is an <a href="/wiki/Abstract_data_type" title="Abstract data type">abstract data type</a> that represents a countable number of ordered <a href="/wiki/Value_(computer_science)" title="Value (computer science)">values</a>, where the same value may occur more than once. An instance of a list is a computer representation of the <a href="/wiki/Mathematics" title="Mathematics">mathematical</a> concept of a finite <a href="/wiki/Sequence_(mathematics)" class="mw-redirect" title="Sequence (mathematics)">sequence</a>; the (potentially) infinite analog of a list is a <a href="/wiki/Stream_(computing)" title="Stream (computing)">stream</a>.<sup id="cite_ref-119" class="reference"><a href="#cite_note-119">&#91;118&#93;</a></sup><sup class="reference" style="white-space:nowrap;">:<span>§3.5</span></sup> Lists are a basic example of <a href="/wiki/Container_(abstract_data_type)" title="Container (abstract data type)">containers</a>, as they contain other values. If the same value occurs multiple times, each occurrence is considered a distinct item.</li>
<li><b><a href="/wiki/Loader_(computing)" title="Loader (computing)">Loader</a></b> – is the part of an <a href="/wiki/Operating_system" title="Operating system">operating system</a> that is responsible for loading <a href="/wiki/Computer_program" title="Computer program">programs</a> and <a href="/wiki/Library_(computing)" title="Library (computing)">libraries</a>. It is one of the essential stages in the process of starting a program, as it places programs into memory and prepares them for execution. Loading a program involves reading the contents of the <a href="/wiki/Executable" title="Executable">executable file</a> containing the program instructions into memory, and then carrying out other required preparatory tasks to prepare the executable for running. Once loading is complete, the operating system starts the program by passing control to the loaded program code.</li>
<li><b><a href="/wiki/Logic_error" title="Logic error">Logic error</a></b> – In <a href="/wiki/Computer_programming" title="Computer programming">computer programming</a>, a logic error  is a <a href="/wiki/Software_bug" title="Software bug">bug</a> in a program that causes it to operate incorrectly, but not to terminate abnormally (or <a href="/wiki/Crash_(computing)" title="Crash (computing)">crash</a>). A logic error produces unintended or undesired output or other behaviour, although it may not immediately be recognized as such.</li>
<li><b><a href="/wiki/Logic_programming" title="Logic programming">Logic programming</a></b> – is a type of <a href="/wiki/Programming_paradigm" title="Programming paradigm">programming paradigm</a> which is largely based on <a href="/wiki/Formal_logic" class="mw-redirect" title="Formal logic">formal logic</a>. Any program written in a logic <a href="/wiki/Programming_language" title="Programming language">programming language</a> is a set of sentences in logical form, expressing facts and rules about some problem domain.  Major logic programming language families include <a href="/wiki/Prolog" title="Prolog">Prolog</a>, <a href="/wiki/Answer_set_programming" title="Answer set programming">answer set programming</a> (ASP) and <a href="/wiki/Datalog" title="Datalog">Datalog</a>.</li>
<li><b><a href="/wiki/Machine_learning" title="Machine learning">Machine learning</a></b> – (ML), is the <a href="/wiki/Branches_of_science" title="Branches of science">scientific study</a> of <a href="/wiki/Algorithm" title="Algorithm">algorithms</a> and <a href="/wiki/Statistical_model" title="Statistical model">statistical models</a> that <a href="/wiki/Computer_systems" class="mw-redirect" title="Computer systems">computer systems</a> use to perform a specific task without using explicit instructions, relying on patterns and <a href="/wiki/Inference" title="Inference">inference</a> instead. It is seen as a subset of <a href="/wiki/Artificial_intelligence" title="Artificial intelligence">artificial intelligence</a>. Machine learning algorithms build a <a href="/wiki/Mathematical_model" title="Mathematical model">mathematical model</a> based on sample data, known as "<a href="/wiki/Training_data" class="mw-redirect" title="Training data">training data</a>", in order to make predictions or decisions without being explicitly programmed to perform the task.<sup id="cite_ref-120" class="reference"><a href="#cite_note-120">&#91;119&#93;</a></sup><sup id="cite_ref-121" class="reference"><a href="#cite_note-121">&#91;120&#93;</a></sup></li>
<li><b><a href="/wiki/Machine_vision" title="Machine vision">Machine vision</a></b> – (MV), is the technology and methods used to provide imaging-based automatic inspection and analysis for such applications as automatic inspection,  <a href="/wiki/Process_control" title="Process control">process control</a>, and robot guidance, usually in industry. Machine vision refers to many technologies, software and hardware products, integrated systems, actions, methods and expertise. Machine vision as a <a href="/wiki/Systems_engineering" title="Systems engineering">systems engineering</a> discipline can be considered distinct from <a href="/wiki/Computer_vision" title="Computer vision">computer vision</a>, a form of <a href="/wiki/Computer_science" title="Computer science">computer science</a>. It attempts to integrate existing technologies in new ways and apply them to solve real world problems. The term is the prevalent one for these functions in industrial automation environments but is also used for these functions in other environments such as security and vehicle guidance.</li>
<li><b><a href="/wiki/Mathematical_logic" title="Mathematical logic">Mathematical logic</a></b> – is a subfield of <a href="/wiki/Mathematics" title="Mathematics">mathematics</a> exploring the applications of formal <a href="/wiki/Logic" title="Logic">logic</a> to mathematics.  It bears close connections to <a href="/wiki/Metamathematics" title="Metamathematics">metamathematics</a>, the <a href="/wiki/Foundations_of_mathematics" title="Foundations of mathematics">foundations of mathematics</a>, and <a href="/wiki/Theoretical_computer_science" title="Theoretical computer science">theoretical computer science</a>.<sup id="cite_ref-122" class="reference"><a href="#cite_note-122">&#91;121&#93;</a></sup>  The unifying themes in mathematical logic include the study of the expressive power of <a href="/wiki/Formal_system" title="Formal system">formal systems</a> and the <a href="/wiki/Deductive_reasoning" title="Deductive reasoning">deductive</a> power of formal <a href="/wiki/Mathematical_proof" title="Mathematical proof">proof</a> systems.</li>
<li><b><a href="/wiki/Matrix_(mathematics)" title="Matrix (mathematics)">Matrix</a></b> – In <a href="/wiki/Mathematics" title="Mathematics">mathematics</a>, a matrix, (plural matrices), is a <a href="/wiki/Rectangle" title="Rectangle">rectangular</a> <i><a href="https://en.wiktionary.org/wiki/array" class="extiw" title="wikt:array">array</a></i><sup id="cite_ref-123" class="reference"><a href="#cite_note-123">&#91;122&#93;</a></sup> (see <a href="/wiki/Irregular_matrix" title="Irregular matrix">irregular matrix</a>) of <a href="/wiki/Number" title="Number">numbers</a>, <a href="/wiki/Symbol_(formal)" title="Symbol (formal)">symbols</a>, or <a href="/wiki/Expression_(mathematics)" title="Expression (mathematics)">expressions</a>, arranged in <i><a href="https://en.wiktionary.org/wiki/row" class="extiw" title="wikt:row">rows</a></i> and <i><a href="https://en.wiktionary.org/wiki/column" class="extiw" title="wikt:column">columns</a></i>.<sup id="cite_ref-124" class="reference"><a href="#cite_note-124">&#91;123&#93;</a></sup><sup id="cite_ref-125" class="reference"><a href="#cite_note-125">&#91;124&#93;</a></sup></li>
<li><b><a href="/wiki/Computer_data_storage" title="Computer data storage">Memory</a></b> – Computer data storage, often called storage, is a technology consisting of <a href="/wiki/Computer" title="Computer">computer</a> components and <a href="/wiki/Data_storage_device" class="mw-redirect" title="Data storage device">recording media</a> that are used to retain digital <a href="/wiki/Data_(computing)" title="Data (computing)">data</a>. It is a core function and fundamental component of computers.<sup id="cite_ref-Patterson_126-0" class="reference"><a href="#cite_note-Patterson-126">&#91;125&#93;</a></sup><sup class="reference" style="white-space:nowrap;">:<span>15–16</span></sup></li>
<li><b><a href="/wiki/Merge_sort" title="Merge sort">Merge sort</a></b> – (also commonly spelled mergesort), is an efficient, general-purpose, <a href="/wiki/Comparison_sort" title="Comparison sort">comparison-based</a> <a href="/wiki/Sorting_algorithm" title="Sorting algorithm">sorting algorithm</a>. Most implementations produce a <a href="/wiki/Sorting_algorithm#Stability" title="Sorting algorithm">stable sort</a>, which means that the order of equal elements is the same in the input and output. Merge sort is a <a href="/wiki/Divide_and_conquer_algorithm" class="mw-redirect" title="Divide and conquer algorithm">divide and conquer algorithm</a> that was invented by <a href="/wiki/John_von_Neumann" title="John von Neumann">John von Neumann</a> in 1945.<sup id="cite_ref-127" class="reference"><a href="#cite_note-127">&#91;126&#93;</a></sup> A detailed description and analysis of bottom-up mergesort appeared in a report by <a href="/wiki/Herman_Goldstine" title="Herman Goldstine">Goldstine</a> and <a href="/wiki/John_von_Neumann" title="John von Neumann">von Neumann</a> as early as 1948.<sup id="cite_ref-128" class="reference"><a href="#cite_note-128">&#91;127&#93;</a></sup></li>
<li><b><a href="/wiki/Method_(computer_programming)" title="Method (computer programming)">Method</a></b> – in <a href="/wiki/Object-oriented_programming" title="Object-oriented programming">object-oriented programming</a> (OOP), is a <a href="/wiki/Procedure_(computer_science)" class="mw-redirect" title="Procedure (computer science)">procedure</a> associated with a <a href="/wiki/Message_passing" title="Message passing">message</a> and an <a href="/wiki/Object_(computer_science)" title="Object (computer science)">object</a>. An object consists of data and behavior. The data and behavior comprise an interface, which specifies how the object may be utilized by any of various consumers<sup id="cite_ref-consumerdef001a_129-0" class="reference"><a href="#cite_note-consumerdef001a-129">&#91;128&#93;</a></sup> of the object.</li>
<li><b><a href="/wiki/Software_development_process" title="Software development process">Methodology</a></b> – In <a href="/wiki/Software_engineering" title="Software engineering">software engineering</a>, a software development process is the process of dividing <a href="/wiki/Software_development" title="Software development">software development</a> work into distinct phases to improve <a href="/wiki/Software_design" title="Software design">design</a>, <a href="/wiki/Software_product_management" title="Software product management">product management</a>, and <a href="/wiki/Software_project_management" title="Software project management">project management</a>. The methodology may include the pre-definition of specific <a href="/wiki/Deliverable" title="Deliverable">deliverables</a> and artifacts that are created and completed by a project team to develop or maintain an application.<sup id="cite_ref-130" class="reference"><a href="#cite_note-130">&#91;129&#93;</a></sup></li>
<li><b><a href="/wiki/Modem" title="Modem">Modem</a></b>- (a <a href="/wiki/Portmanteau" title="Portmanteau">portmanteau</a> of modulator-demodulator) is a <a href="/wiki/Computer_hardware" title="Computer hardware">hardware</a> device that converts data into a format suitable for a <a href="/wiki/Transmission_medium" title="Transmission medium">transmission medium</a> so that it can be transmitted from one computer to another (historically along telephone wires). A modem <a href="/wiki/Modulation#Digital_modulation_methods" title="Modulation">modulates</a> one or more <a href="/wiki/Carrier_wave" title="Carrier wave">carrier wave</a> signals to encode <a href="/wiki/Digital_information" class="mw-redirect" title="Digital information">digital information</a> for transmission and <a href="/wiki/Demodulation" title="Demodulation">demodulates</a> signals to decode the transmitted information. The goal is to produce a <a href="/wiki/Signal_(electronics)" class="mw-redirect" title="Signal (electronics)">signal</a> that can be transmitted easily and decoded reliably to reproduce the original digital data. Modems can be used with almost any means of transmitting analog signals from <a href="/wiki/Light-emitting_diode" title="Light-emitting diode">light-emitting diodes</a> to <a href="/wiki/Radio" title="Radio">radio</a>. A common type of modem is one that turns the <a href="/wiki/Digital_data" title="Digital data">digital data</a> of a <a href="/wiki/Computer" title="Computer">computer</a> into modulated <a href="/wiki/Electrical_signal" class="mw-redirect" title="Electrical signal">electrical signal</a> for transmission over <a href="/wiki/Telephone_line" title="Telephone line">telephone lines</a> and demodulated by another modem at the receiver side to recover the digital data.</li>

Challenges in natural language processing frequently involve <a href="/wiki/Speech_recognition" title="Speech recognition">speech recognition</a>, <a href="/wiki/Natural_language_understanding" class="mw-redirect" title="Natural language understanding">natural language understanding</a>, and <a href="/wiki/Natural_language_generation" class="mw-redirect" title="Natural language generation">natural language generation</a>.</li>

<li><b><a href="/wiki/Object_(computer_science)" title="Object (computer science)">Object</a></b> – An object can be a <a href="/wiki/Variable_(computer_science)" title="Variable (computer science)">variable</a>, a <a href="/wiki/Data_structure" title="Data structure">data structure</a>, a <a href="/wiki/Subroutine" title="Subroutine">function</a>, or a <a href="/wiki/Method_(computer_programming)" title="Method (computer programming)">method</a>, and as such, is a <a href="/wiki/Value_(computer_science)" title="Value (computer science)">value</a> in <a href="/wiki/Memory_address" title="Memory address">memory</a> referenced by an <a href="/wiki/Identifier_(computer_programming)" class="mw-redirect" title="Identifier (computer programming)">identifier</a>.  In the <a href="/wiki/Class-based_programming" title="Class-based programming">class-based</a> <a href="/wiki/Object-oriented_programming" title="Object-oriented programming">object-oriented programming</a> paradigm, <i>object</i> refers to a particular <a href="/wiki/Instance_(computer_science)" title="Instance (computer science)">instance</a> of a <a href="/wiki/Class_(computer_science)" class="mw-redirect" title="Class (computer science)">class</a>, where the object can be a combination of variables, functions, and data structures.  In <a href="/wiki/Relational_model" title="Relational model">relational</a> <a href="/wiki/Database" title="Database">database</a> management, an object can be a table or column, or an association between data and a database entity (such as relating a person's age to a specific person).<sup id="cite_ref-Oppel_131-0" class="reference"><a href="#cite_note-Oppel-131">&#91;130&#93;</a></sup></li>
<li><b><a href="/wiki/Object_code" title="Object code">Object code</a></b> – or object module, is the product of a <a href="/wiki/Compiler" title="Compiler">compiler</a>.<sup id="cite_ref-132" class="reference"><a href="#cite_note-132">&#91;131&#93;</a></sup> In a general sense object code is a sequence of <a href="/wiki/Statement_(computer_science)" title="Statement (computer science)">statements</a> or instructions in a computer language,<sup id="cite_ref-133" class="reference"><a href="#cite_note-133">&#91;132&#93;</a></sup> usually a <a href="/wiki/Machine_code" title="Machine code">machine code</a> language (i.e., <a href="/wiki/Binary_file" title="Binary file">binary</a>) or an intermediate language such as <a href="/wiki/Register_transfer_language" title="Register transfer language">register transfer language</a> (RTL). <li><b><a href="/wiki/Object_oriented_programming" class="mw-redirect" title="Object oriented programming">Object oriented programming</a></b> – (OOP),  is a <a href="/wiki/Programming_paradigm" title="Programming paradigm">programming paradigm</a> based on the concept of "<a href="/wiki/Object_(computer_science)" title="Object (computer science)">objects</a>", which can contain <a href="/wiki/Data" title="Data">data</a>, in the form of <a href="/wiki/Field_(computer_science)" title="Field (computer science)">fields</a> (often known as <i>attributes</i> or <i>properties</i>), and code, in the form of procedures (often known as <i><a href="/wiki/Method_(computer_science)" class="mw-redirect" title="Method (computer science)">methods</a></i>). A feature of objects is an object's procedures that can access and often modify the data fields of the object with which they are associated (objects have a notion of "<a href="/wiki/This_(computer_programming)" title="This (computer programming)">this</a>" or "self"). In OOP, computer programs are designed by making them out of objects that interact with one another.<sup id="cite_ref-134" class="reference"><a href="#cite_note-134">&#91;133&#93;</a></sup><sup id="cite_ref-135" class="reference"><a href="#cite_note-135">&#91;134&#93;</a></sup> OOP languages are diverse, but the most popular ones are <a href="/wiki/Class-based_programming" title="Class-based programming">class-based</a>, meaning that objects are <a href="/wiki/Instance_(computer_science)" title="Instance (computer science)">instances</a> of <a href="/wiki/Class_(computer_science)" class="mw-redirect" title="Class (computer science)">classes</a>, which also determine their <a href="/wiki/Data_type" title="Data type">types</a>.</li>

<li><b><a href="/wiki/Optical_fiber" title="Optical fiber">Optical fiber</a></b> –  is a flexible, <a href="/wiki/Transparency_and_translucency" title="Transparency and translucency">transparent</a> <a href="/wiki/Fiber" title="Fiber">fiber</a> made by <a href="/wiki/Drawing_(manufacturing)" title="Drawing (manufacturing)">drawing</a> <a href="/wiki/Glass" title="Glass">glass</a> (<a href="/wiki/Silica" class="mw-redirect" title="Silica">silica</a>) or plastic to a diameter slightly thicker than that of a <a href="/wiki/Hair%27s_breadth" title="Hair&#39;s breadth">human hair</a>.<sup id="cite_ref-138" class="reference"><a href="#cite_note-138">&#91;137&#93;</a></sup> Optical fibers are used most often as a means to transmit light between the two ends of the fiber and find wide usage in <a href="/wiki/Fiber-optic_communication" title="Fiber-optic communication">fiber-optic communications</a>, where they permit transmission over longer distances and at higher <a href="/wiki/Bandwidth_(computing)" title="Bandwidth (computing)">bandwidths</a> (data rates) than electrical cables. Fibers are used instead of <a href="/wiki/Metal" title="Metal">metal</a> wires because signals travel along them with less <a href="/wiki/Attenuation" title="Attenuation">loss</a>; in addition, fibers are immune to <a href="/wiki/Electromagnetic_interference" title="Electromagnetic interference">electromagnetic interference</a>, a problem from which metal wires suffer.<sup id="cite_ref-139" class="reference"><a href="#cite_note-139">&#91;138&#93;</a></sup></li>

An ordered list of parameters is usually included in the <a href="/wiki/Function_signature" class="mw-redirect" title="Function signature">definition of a subroutine</a>, so that, each time the subroutine is called, its arguments for that call are evaluated, and the resulting values can be assigned to the corresponding parameters.</li>

<li><b><a href="/wiki/Priority_queue" title="Priority queue">Priority queue</a></b> – is an <a href="/wiki/Abstract_data_type" title="Abstract data type">abstract data type</a> which is like a regular <a href="/wiki/Queue_(abstract_data_type)" title="Queue (abstract data type)">queue</a> or <a href="/wiki/Stack_(abstract_data_type)" title="Stack (abstract data type)">stack</a> data structure, but where additionally each element has a "priority" associated with it. <li><b><a href="/wiki/Regression_testing" title="Regression testing">Regression testing</a></b> –  (rarely <i>non-regression testing</i><sup id="cite_ref-148" class="reference"><a href="#cite_note-148">&#91;146&#93;</a></sup>) is re-running <a href="/wiki/Functional_testing" title="Functional testing">functional</a> and <a href="/wiki/Non-functional_testing" title="Non-functional testing">non-functional tests</a> to ensure that previously developed and tested software still performs after a change.<sup id="cite_ref-149" class="reference"><a href="#cite_note-149">&#91;147&#93;</a></sup> If not, that would be called a <i><a href="/wiki/Software_regression" title="Software regression">regression</a></i>. <li><b><a href="/wiki/Variable_(computer_science)" title="Variable (computer science)">Variable</a></b> – In <a href="/wiki/Computer_programming" title="Computer programming">computer programming</a>, a variable, or scalar, is a storage location (identified by a <a href="/wiki/Memory_address" title="Memory address">memory address</a>) paired with an associated <a href="/wiki/Symbol" title="Symbol">symbolic name</a> (an <i><a href="/wiki/Identifier_(computer_programming)" class="mw-redirect" title="Identifier (computer programming)">identifier</a></i>), which contains some known or unknown quantity of information referred to as a <i><a href="/wiki/Value_(computer_science)" title="Value (computer science)">value</a></i>. <li><b><a href="/wiki/Waterfall_model" title="Waterfall model">Waterfall model</a></b> – is a breakdown of project activities into linear <a href="/wiki/Sequence" title="Sequence">sequential</a> phases, where each phase depends on the deliverables of the previous one and corresponds to a specialisation of tasks. <li><b><a href="/wiki/WAV" title="WAV">Waveform Audio File Format</a></b> – (WAVE, or WAV due to its <a href="/wiki/Filename_extension" title="Filename extension">filename extension</a>), is an <a href="/wiki/Audio_file_format" title="Audio file format">audio file format</a> standard, developed by <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> and <a href="/wiki/International_Business_Machines" class="mw-redirect" title="International Business Machines">IBM</a>, for storing an audio bitstream on <a href="/wiki/Personal_computer" title="Personal computer">PCs</a>. It is the main format used on <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a> systems for raw and typically uncompressed audio. The usual bitstream encoding is the <a href="/wiki/Linear_pulse-code_modulation" class="mw-redirect" title="Linear pulse-code modulation">linear pulse-code modulation</a> (LPCM) format.</li>
